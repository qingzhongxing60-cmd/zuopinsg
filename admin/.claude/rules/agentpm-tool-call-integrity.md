# 工具调用完整性

## 铁律

```
调用工具时，必须发起真正的结构化工具调用，
严禁把工具调用输出成 call / invoke 之类的纯文本。
```

## 问题描述

错误表现：本该发起工具调用（如 Read / Write / Edit / Bash），却把调用写成了纯文本，例如直接打出 `call`、`invoke`、`课` 等字样，导致：

- 工具根本没执行（文件没创建、命令没运行、内容没读取）
- 用户看到一堆无意义文本，体验割裂
- 流程卡住，需要重新发起调用，浪费往返

## 强制要求

1. **每次要操作工具时，发起真正的工具调用，不输出任何代表工具调用的纯文本**
2. **不在正文里打出 `call` / `invoke` 当作"我要调用工具"的占位**
3. 工具调用前的说明文字保持简短，说完立即发起真正的调用，不要用文本"模拟"调用动作
4. 如果发现自己上一轮把工具调用打成了文本（工具未执行），立即用真正的工具调用补上，不要重复打文本

## 自检

每次准备操作工具前自问：
- 我接下来是要**真正执行**这个工具，还是只是在描述？
- 要执行 → 发起结构化工具调用（不是文本）
- 只是描述 → 用正常中文说明，不写 `call`/`invoke`

## 长会话提示

会话越长、上下文越大，越容易出现"该调用工具却输出文本"的失稳。
出现此现象频繁时，可提示用户新开会话以恢复工具调用稳定性。
