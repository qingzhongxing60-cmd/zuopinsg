# 编码规范

## 克制原则

只做被要求的事，不要自作主张添加额外内容。需求里没有提到的视觉效果、交互效果、样式属性，一律不加。

## 硬性规则

1. 单文件行数 ≤ 500 行；单个函数/方法 ≤ 80 行
2. 禁止 `console.log`，调试日志必须在提交前清除
3. 禁止裸写 `fetch` / `XMLHttpRequest`，统一通过项目封装的 `request` 工具
4. 禁止在组件内直接操作 DOM，使用 Vue 响应式或 `ref`
5. Mock 数据变量必须在模块顶层声明
6. 新增页面必须同步添加路由
7. API 函数必须有 JSDoc 注释

## 注释规范

- Vue 组件：`<script setup>` 顶部写组件用途（1-2行），功能块前加单行注释
- TypeScript：导出函数/接口前写 JSDoc
- CSS：非显而易见的样式值加注释说明原因

## 大块内容写入

超过 80 行的内容禁止一次性写入，必须分块：
- Write：先写前 50 行，用 Edit 追加剩余（每次 ≤ 50 行）
- Edit 替换：占位符接力法（`<!-- PART_N -->`）

## 并发操作

同一条消息内，所有无依赖关系的操作必须并行执行。

## 配置保护

禁止修改 `.eslintrc.*`、`.prettierrc.*`、`tsconfig.json`、`vite.config.*` 来消除报错。修代码，不改规则。

## 系统边界校验

用户输入、外部 API 返回值、URL 参数等外部数据，必须在使用前校验。内部模块间调用无需重复校验。
