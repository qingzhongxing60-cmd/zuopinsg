# 详细设计（LLD）内容规范

> 适用于 lld-design 技能与 design-analyzer / design-writer / design-reviewer 三代理。
> 详细设计回答"每个模块如何实现、数据如何落库、接口如何定义"，是开发可直接照着编码的依据。

## 目录

- 一、核心定位与三大维度
- 二、引用的技术规范（强制遵循）
- 三、维度一：模块详细设计
- 四、维度二：数据库物理设计
- 五、维度三：API 详细设计
- 六、安全设计
- 七、质量清单

## 一、核心定位与三大维度

```
HLD（系统怎么搭） → LLD（每块怎么实现） → 编码
```

LLD 的读者是开发工程师。一份合格的 LLD 应让开发不需要再做设计决策，直接编码。每个功能模块的 LLD 包含三大维度：

| 维度 | 回答 | 产出 |
|------|------|------|
| 模块详细设计 | 这个模块内部怎么组织、核心逻辑怎么走 | 类图、时序图、关键逻辑、异常处理 |
| 数据库物理设计 | 数据怎么落库 | 表结构、字段、索引、关系、DDL |
| API 详细设计 | 接口怎么调 | 接口清单、请求/响应、错误码、鉴权 |

**LLD 的每张表、每个接口、每个类必须可追溯到 HLD 模块或 SRS 功能。**

## 二、引用的技术规范（强制遵循）

LLD 不重新发明规范，直接遵循项目已有技术规范。design-writer / design-reviewer 必须读取：

| 规范文件 | 约束内容 |
|---------|---------|
| `phase3-development/database.md` | 表/字段/索引/外键命名、必备字段、索引策略、软删除 |
| `phase3-development/backend.md` | RESTful 路由、统一响应格式、分层模式、错误处理 |
| `conventions/security.md` | 认证授权、权限控制、数据校验、敏感数据处理 |

**冲突时以技术规范为准**，LLD 内容必须与这些规范一致（如表必须有 id/created_at/updated_at）。

## 三、维度一：模块详细设计

每个模块包含：

1. **模块职责与边界**：一句话职责 + 边界，标注追溯（HLD 模块 / SRS 功能编号）
2. **核心类/服务设计**：
   - 类清单表（类/服务名 / 职责 / 关键方法）
   - **类图**（diagram-generator，class 类型）：展示类、属性、方法、类间关系
   - 遵循 backend.md 分层（Controller → Service → Repository）
3. **核心业务流程**：
   - 选取模块的关键操作（如"创建运单""审核工单"）
   - **时序图**（diagram-generator，sequence 类型）：展示调用链
   - 配文字步骤：含参数校验、事务边界、分支判断、跨模块调用
   - 关键逻辑可用伪代码描述（描述步骤，不抄业务源码）
4. **状态机**（仅有明确状态流转的模块）：状态枚举 + 流转触发条件 + 流转动作 + 状态流转图（diagram-generator，flowchart 类型）
5. **异常处理**：异常场景 / 处理策略 / 错误反馈，遵循 backend.md 统一错误类

## 四、维度二：数据库物理设计

**这是 LLD 最关键、最易出错的维度。**

### 每张表的内容结构

1. **表说明**：表名（snake_case 复数）+ 业务含义
2. **字段清单表**（逐字段，不省略），固定六列：

| 字段名 | 类型 | 允许空 | 键 | 默认值 | 说明 |
|-------|------|-------|----|-------|------|
| id | INT AUTO_INCREMENT | 否 | PK | - | 主键ID |
| plate_no | VARCHAR(20) | 否 | UNI | - | 车牌号 |
| status | TINYINT | 否 | - | 1 | 状态：1-启用 0-禁用 |
| created_at | DATETIME | 否 | - | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | 否 | - | CURRENT_TIMESTAMP | 更新时间 |
| created_by | INT | 是 | - | NULL | 创建人ID |
| updated_by | INT | 是 | - | NULL | 更新人ID |

3. **索引清单表**：索引名 / 字段 / 类型（普通/唯一/复合）/ 说明
4. **关系说明**：外键 / 本表字段 / 关联表 / 关联字段 / 关系（用于物理 ER 图）
5. **建表 DDL**（可选但推荐）：SQL 代码块，必须与字段表一致

### 物理 ER 图
整个数据库章节配一张**物理 ER 图**（diagram-generator，er 类型），含表、主键、外键关系。

### 硬性规则（来自 database.md）

| 规则 | 要求 |
|------|------|
| 必备字段 | 每张表必须有 id / created_at / updated_at / created_by / updated_by |
| 表名 | snake_case，复数（registered_users） |
| 字段名 | snake_case（created_at） |
| 索引名 | idx_表名_字段名 |
| 外键名 | fk_表名_关联表名 |
| 外键索引 | 外键字段必须建索引 |
| 索引数量 | 单表 ≤ 5 个 |
| 低区分度字段 | status/gender 等不单独建索引 |
| 删除策略 | 优先软删除（status 标记），不物理删除 |
| 枚举字段 | 说明列写明取值含义 |

### mock 转表结构注意
前端 mock 的 interface 是视图模型，缺少持久化字段。转换时：
- 业务字段从 mock/entity 提取，标注来源
- 补全 id/created_at 等必备字段，标注 [规范补全]
- 关联字段（如选司机带出手机号）：手机号属司机表，不在本表冗余存储，除非有冗余设计理由
- 设计阶段新增的表（中间表、字典表）标注 [设计补全]

## 五、维度三：API 详细设计

### 接口清单（先总览）

| 方法 | 路径 | 用途 | 鉴权/权限码 |
|------|------|------|-----------|
| GET | /api/waybills | 运单列表 | waybill:list |
| POST | /api/waybills | 创建运单 | waybill:create |

### 每个接口的明细

1. **基本信息**：接口名 / 方法+路径 / 用途 / 鉴权要求
2. **请求参数表**：参数 / 位置（path/query/body）/ 类型 / 必填 / 说明 / 校验规则
3. **成功响应**：JSON 示例，套用 backend.md 统一响应格式

```json
{ "code": 0, "success": true, "message": "成功", "data": {}, "timestamp": 0 }
```
列表接口 data 用 PageResponse：`{ "list": [], "total": 0, "page": 1, "pageSize": 20 }`

4. **错误码表**：场景 / HTTP状态码 / 业务code / message
5. **事务/幂等说明**：涉及多表写入或重复提交风险时必写

### 硬性规则（来自 backend.md）

| 规则 | 要求 |
|------|------|
| 路由 | RESTful：GET列表/单条、POST创建、PUT/PATCH更新、DELETE删除 |
| 响应格式 | 统一 ApiResponse<T>，列表用 PageResponse<T> |
| 错误处理 | 错误码不泄露内部信息，统一错误类 |
| 输入验证 | 每个接口标注参数校验规则 |
| 鉴权 | 写操作接口必须标注权限码 |

## 六、安全设计

对照 security.md，每个模块说明：
- 接口鉴权方式（JWT / Session）与所需权限码
- 输入校验（防注入、参数白名单）
- 敏感数据处理（加密存储、传输 HTTPS、日志脱敏）
- 越权防护（数据级权限，如只能查本部门数据）

## 七、质量清单

每个 LLD 模块写完后对照：

**模块详细设计：**
- [ ] 模块职责标注了 HLD/SRS 追溯
- [ ] 有类清单 + 类图
- [ ] 核心流程有时序图 + 文字步骤（含校验、事务、分支）
- [ ] 异常处理覆盖主要异常场景

**数据库物理设计：**
- [ ] 每张表有 id / created_at / updated_at / created_by / updated_by
- [ ] 表名 snake_case 复数，字段 snake_case
- [ ] 字段数 = 业务字段 + 必备字段，逐个列出无省略
- [ ] 外键字段都建了索引，索引名 idx_表_字段，外键名 fk_表_关联表
- [ ] 单表索引 ≤5，未在低区分度字段单独建索引
- [ ] 枚举字段说明了取值含义
- [ ] DDL 与字段表一致
- [ ] 有物理 ER 图
- [ ] [设计补全]/[规范补全] 已标注

**API 详细设计：**
- [ ] 有接口清单总览
- [ ] 每个接口有请求参数表 + 成功响应 + 错误码表
- [ ] 路径符合 RESTful，响应套用统一格式
- [ ] 写操作接口标注了鉴权/权限码
- [ ] 涉及多表写入的接口说明了事务

**可追溯性：**
- [ ] 每张表/每个接口/每个类可追溯到 HLD 模块或 SRS 功能
