# 标注 content 编写规范

## 核心原则

**content 要原样呈现需求文档的信息，不要改写成自己的描述。有表格就用表格，有文案就用原文。**

相关规则：
- 什么该标、什么不该标 → 见 `strategy.md`
- class 命名和注入位置 → 见 `selector-patterns.md`

---

## 1. 字段类区域（筛选区、列表、表单）

**必须使用 Markdown 表格**，直接从需求文档搬过来，不要改写成文字描述。

### 筛选区格式

```markdown
| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 设备名称 | 文本框 | 模糊搜索 |
| 所属单位 | 树形选择 | 数据源：/department/tree，支持搜索 |
| 设备状态 | 下拉 | 枚举：正常/停用/维修中 |

点击搜索按钮触发查询，与左侧分类树联动叠加过滤。
```

### 列表字段格式

```markdown
| 列名 | 类型 | 说明 |
|-----|------|------|
| 设备名称 | 图片+文本 | 左侧缩略图，点击预览；无图显示占位图，无图不触发预览 |
| 设备状态 | 标签 | 正常=绿色 / 停用=灰色 / 维修中=橙色 |
| 关联厂商 | 文本 | 无值显示 - |

分页默认每页10条。支持批量勾选，勾选后显示批量删除按钮。空数据提示：暂无xxx数据。
```

### 表单字段格式

```markdown
| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 设备名称 | 文本框 | 2-50字 | 是 | - | - |
| 设备编号 | 文本框 | 2-30字 | 是 | - | 系统内唯一 |
| 设备状态 | 下拉 | - | 是 | 正常 | 正常/停用/维修中 |
| 设备图片 | 图片上传 | 最多5张 | 否 | - | 单张最大10MB |

提交失败时保留填写内容并显示具体错误原因。
```

## 2. 操作类区域（操作栏、导入弹窗）

列出所有操作及其行为，**确认文案和提示文案用原文**。

```markdown
左侧按钮：新增（打开右侧抽屉）、导入（打开导入弹窗）、批量删除（勾选后显示，含已选数量）。
右侧按钮：导出（导出当前筛选结果为 Excel，提示：导出成功）。
```

## 3. 业务规则类

**触发条件 + 影响范围 + 错误提示原文**，三要素缺一不可。

```markdown
触发条件：新增/编辑时设备编号与已有记录重复。
处理方式：后端阻止提交，前端展示错误提示。
错误提示原文：设备编号【XXX】已存在，请更换。
```

---

## 4. 禁止写法

- ❌ "支持搜索" → ✅ "数据源：/vendor/options，支持关键词搜索"
- ❌ "点击新增按钮可以新增" → ✅ 直接写表单字段表格
- ❌ 把表格改写成文字描述 → ✅ 保留 Markdown 表格格式
- ❌ 省略数据来源接口路径 → ✅ 写明接口路径
- ❌ 省略提示文案 → ✅ 写原文文案
- ❌ 把角色名/权限说明写进标注（如"仅系统管理员可见"、"业务主办可操作"） → ✅ 标注只说功能行为；角色权限由业务配置权限码控制，不在标注里写死
