# SRS 需求规格说明书质量审查清单

> 供 req-reviewer agent 使用。逐项检查，每项必须明确标注 ✅ 通过 / ❌ 不通过（含具体问题位置）。
> **禁止笼统说"全部通过"，每项必须逐一标注结果。**

---

## 维度一：语言规范

### 1.1 禁止词汇扫描（对照 prd-language.md）

- [ ] 无 UI 组件名（弹窗、Modal、Drawer、抽屉、a-xxx、el-xxx、van-xxx 等）
- [ ] 无接口路径（/api/xxx 格式）
- [ ] 无 CSS 属性（flex、grid、margin、px 等）
- [ ] 无技术实现词（调用接口、API 请求、前端过滤、回填、回显等）
- [ ] 无口语化表达

### 1.2 数据来源描述

- [ ] 下拉/选择字段的数据来源使用业务语言（"来源于[模块]中已启用的[数据]"，不写接口路径）

### 1.3 业务术语一致性

- [ ] 启用/停用（不混用"正常/禁用"、"开启/关闭"）
- [ ] 新增/编辑/删除（不混用"添加/修改/移除"）
- [ ] 与项目已有术语表一致（如有）

---

## 维度二：章节结构完整性

### 2.1 五个子章节是否存在

- [ ] （一）功能需求概述：存在，1-3句话，不含技术词汇
- [ ] （二）功能设计：存在，按子功能逐一展开
- [ ] （三）业务逻辑关联：存在，有跨模块依赖说明
- [ ] （四）用户操作流程图：存在，有图片引用占位
- [ ] （五）功能时序图：存在，有图片引用占位

### 2.2 字段表格列完整性

- [ ] 筛选表格：字段名 / 类型 / 说明（三列）
- [ ] 列表表格：列名 / 字段类型 / 说明（三列），含分页说明和空数据提示
- [ ] 表单表格（新增/编辑）：字段名称 / 类型 / 长度限制 / 必填 / 默认值 / 说明（六列）

### 2.3 各子功能描述完整性

- [ ] 新增功能：有字段表格 + 提交成功提示 + 提交失败说明
- [ ] 编辑功能：有字段说明（同新增或单独列出）+ 提交成功提示
- [ ] 删除功能：有确认文案原文 + 成功提示 + 关联保护说明（如有）
- [ ] 启用/停用功能：有确认文案（停用）+ 成功提示
- [ ] 导出功能：有成功提示 + 失败提示（如有异步生成，有等待提示）

### 2.4 多端系统额外检查

- [ ] 若系统包含移动端，移动端章节（3.5.x 移动端）是否存在
- [ ] 移动端章节是否按 Tab 结构组织（每个 Tab 对应一个子章节）
- [ ] 3.3 页面功能清单是否包含终端列，PC 端和移动端是否分开列出

---

## 维度三：功能描述三要素

每个功能模块必须同时具备：**数据定义 + 业务规则 + 交互功能**

### 3.1 数据定义

- [ ] 所有字段有明确的类型定义
- [ ] 枚举值有完整列举（不能只写"下拉选择"而不列枚举值）
- [ ] 只读字段标注了数据来源

### 3.2 业务规则（每条必须包含三要素）

- [ ] 触发条件：明确说明什么情况下触发
- [ ] 影响范围：说明影响哪些字段/模块/状态
- [ ] 提示文案原文：必须是具体文案，不能写"提示错误"等模糊描述

### 3.3 交互功能

- [ ] 操作入口有描述（点击什么按钮/如何进入）
- [ ] 操作结果有描述（成功后发生什么）
- [ ] 异常场景有描述（失败/权限不足/空数据时发生什么）

---

## 维度四：提示文案完整性

### 4.1 操作结果提示

- [ ] 新增成功提示：有具体文案原文
- [ ] 编辑成功提示：有具体文案原文
- [ ] 删除确认文案：有具体文案原文（"确定要删除...吗？"格式）
- [ ] 删除成功提示：有具体文案原文
- [ ] 启用成功提示：有具体文案原文
- [ ] 停用确认文案：有具体文案原文
- [ ] 停用成功提示：有具体文案原文

### 4.2 校验错误提示

- [ ] 必填字段为空时的提示（"请填写【字段名】"）
- [ ] 唯一性校验错误提示（如有唯一性约束）
- [ ] 格式校验错误提示（如有格式要求）
- [ ] 关联保护阻止提示（如有关联数据保护）

### 4.3 边界场景提示

- [ ] 空数据提示：列表无数据时的提示文案
- [ ] 权限不足处理：无权限时按钮完全隐藏（不渲染，非禁用）
- [ ] 导出成功/失败提示（如有导出功能）
- [ ] 网络中断提示（移动端功能如有离线操作）

---

## 维度五：内容分级合规

> 对照 prd-language.md 的「内容分级规则」，检查各章节内容是否放在正确的层级。

### 5.1 L1 概览层检查（3.2、3.3）

- [ ] 无像素值、颜色代码、CSS 属性
- [ ] 无图表视觉类型（折线图/柱状图/饼图/圆环图）
- [ ] 无布局位置描述（左侧/右侧/顶部/底部）
- [ ] 无组件数量+视觉描述组合（"4 张 KPI 卡"）
- [ ] 格式为"动词+对象+条件"

### 5.2 L2 行为层检查（3.5.x 功能设计主体，排除「视觉与交互规格」子节）

- [ ] 无像素值（Npx、Nrem）
- [ ] 无颜色代码（#xxx、rgb、rgba）
- [ ] 无动画描述（cubic-bezier、淡入、滑出、ease-in、ease-out）
- [ ] 无 CSS 属性名（border-radius、box-shadow、backdrop-filter）
- [ ] 无字体具体数值（字号 Npx、font-weight 数值）
- [ ] 无视觉效果词（毛玻璃、渐变、阴影）
- [ ] 视觉细节如有，集中在独立的「视觉与交互规格」子节

### 5.3 正则扫描清单

审查时对 L1/L2 内容执行以下正则匹配，命中即标记 ❌：

| 正则 | 检测目标 |
|------|---------|
| `\d+px` / `\d+rem` | 尺寸值 |
| `#[0-9a-fA-F]{3,8}` | 颜色代码 |
| `rgb\(` / `rgba\(` | RGB 颜色 |
| `cubic-bezier` / `ease-in` / `ease-out` | 动画函数 |
| `border-radius` / `box-shadow` / `backdrop-filter` | CSS 属性 |
| `font-weight` / `font-size` | 字体属性 |
| `毛玻璃` / `渐变` / `阴影`（在 L2 中） | 视觉效果词 |
| `折线图` / `柱状图` / `饼图` / `圆环图`（在 L1 中） | 图表视觉类型 |
