# 用户操作流程图绘图规范

> 适用于：单功能操作流程、用户交互流程、审批流程。
> 前置依赖：必须先读取 `common-rules.md` 通用规范。
> 模板目录：`examples/operation-flow/`

---

## 一、与业务流程图的区别

| 维度 | 业务流程图 | 操作流程图 |
|------|-----------|-----------|
| 布局 | 泳道（多角色分区） | 纯流程图（无泳道） |
| 容器 | 外层 table + 子泳道 | 无容器，节点直接放 parent="1" |
| 流程范围 | 端到端业务全流程 | 单功能内的操作步骤 |
| 角色表达 | 泳道标题区分角色 | 不区分角色，聚焦操作步骤 |
| 连线风格 | 正交连线（orthogonal） | 正交连线（orthogonal） |

---

## 二、布局规则

### 2.1 无容器、无泳道

操作流程图不使用任何容器或泳道。所有节点和连线的 parent 直接指向 `"1"`（根节点）。

```xml
<mxCell id="0"/>
<mxCell id="1" parent="0"/>
<!-- 所有节点 parent="1" -->
<mxCell id="n_start" parent="1" vertex="1" .../>
<mxCell id="n_step1" parent="1" vertex="1" .../>
<mxCell id="e1" parent="1" edge="1" source="n_start" target="n_step1" .../>
```

### 2.2 流程方向

主流程从上到下（垂直方向），分支向左右展开：

```
        [开始]
          ↓
        [步骤1]
          ↓
       <判断?>
      ↙       ↘
  [分支A]   [分支B]
      ↘       ↙
        [步骤N]
          ↓
        [结束]
```

### 2.3 画布起始坐标

- 主流程居中，起始 X 根据画布宽度计算
- 起始 Y = 40px
- 推荐画布宽度：600-800px（单列流程）

## 三、节点规则

### 3.1 节点类型与样式

**开始节点（圆角椭圆，蓝色）：**
```
ellipse;whiteSpace=wrap;html=1;fillColor=#dae8fc;strokeColor=#6c8ebf;fontSize=12;
尺寸：width=120, height=40
```

**操作步骤（圆角矩形，绿色）：**
```
rounded=1;whiteSpace=wrap;html=1;fillColor=#d5e8d4;strokeColor=#82b366;fontSize=12;
尺寸：width=140, height=40
```

**判断节点（菱形，橙色）：**
```
rhombus;whiteSpace=wrap;html=1;fillColor=#fff2cc;strokeColor=#d6b656;fontSize=12;
尺寸：width=120, height=60
```

**结束节点（圆角椭圆，蓝色）：**
```
ellipse;whiteSpace=wrap;html=1;fillColor=#dae8fc;strokeColor=#6c8ebf;fontSize=12;
尺寸：width=120, height=40
```

### 3.2 节点尺寸

- 所有操作步骤节点宽高一致（同类型统一尺寸）
- 菱形判断节点宽度可略大（容纳文字）
- 开始/结束节点尺寸一致

---

## 四、间距规则

### 4.1 垂直间距

- 相邻节点垂直间距：≥ 50px（节点底部到下一节点顶部）
- 判断节点到分支节点：≥ 60px

### 4.2 水平间距

- 分支节点与主流程节点水平间距：≥ 80px
- 并列分支节点间水平间距：≥ 60px

### 4.3 对齐

- 主流程节点水平居中对齐（同一 X 中心线）
- 左分支节点对齐到同一 X
- 右分支节点对齐到同一 X

---

## 五、连线规则

### 5.1 连线样式

所有连线使用正交路由 + 箭头：
```
edgeStyle=orthogonalEdgeStyle;rounded=0;
```

### 5.2 主流程连线

上下相邻节点的连线：
```
exitX=0.5;exitY=1（底部出）
entryX=0.5;entryY=0（顶部入）
```

### 5.3 判断分支连线

从判断节点出发：
- 向下（默认/是）：`exitX=0.5;exitY=1`
- 向左（否/分支A）：`exitX=0;exitY=0.5`
- 向右（分支B）：`exitX=1;exitY=0.5`

分支连线必须带标签（"是"/"否" 或具体条件）。

### 5.4 回路连线

循环回路从分支节点右侧出，向上回到判断节点右侧入：
```
exitX=1;exitY=0.5 → entryX=1;entryY=0.5
```
使用 waypoints 控制回路路径，避免穿越其他节点。

### 5.5 汇聚连线

多分支汇聚到同一节点时，从不同方向进入：
- 左分支从左侧入：`entryX=0;entryY=0.5`
- 右分支从右侧入：`entryX=1;entryY=0.5`
- 主流程从顶部入：`entryX=0.5;entryY=0`

---

## 六、禁止行为

1. **禁止使用泳道** — 操作流程图不需要角色分区
2. **禁止使用容器** — 节点直接放在根节点下
3. **禁止无标签的判断分支** — 每条从菱形出发的线必须有标签
4. **禁止悬空节点** — 每个节点必须有入线和出线（开始/结束除外）
5. **禁止连线穿越节点** — 回路和汇聚线必须绕开其他节点

## 七、自检清单

生成 XML 后、输出前，逐项检查：

- [ ] 无泳道、无容器（搜索 "swimlane" 应为 0）
- [ ] 所有节点 parent="1"
- [ ] 主流程节点水平居中对齐
- [ ] 判断节点每条出线都有标签
- [ ] 相邻节点垂直间距 ≥ 50px
- [ ] 连线不穿越其他节点
- [ ] 回路连线使用 waypoints 绕行
- [ ] 开始和结束节点各只有一个
