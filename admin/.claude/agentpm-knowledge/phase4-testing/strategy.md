# 测试阶段规范

## 测试类型与覆盖要求

| 测试类型 | 覆盖目标 | 最低要求 |
|---------|---------|---------|
| 单元测试 | 工具函数、业务逻辑 | 核心模块 100% |
| 集成测试 | API 端点、数据库操作 | 所有接口 |
| E2E 测试 | 关键用户流程 | 主流程全覆盖 |
| 边界测试 | 边界值、异常输入 | 每个功能点 |
| 权限测试 | 角色权限控制 | 所有权限点 |

整体覆盖率目标：**≥ 80%**

## 测试用例设计原则

每个功能点的测试用例必须覆盖：

1. **正常流程**：标准输入，期望输出
2. **边界值**：最大值、最小值、空值、零值
3. **异常场景**：非法输入、网络错误、权限不足
4. **并发场景**：重复提交、并发操作（如有）

## 验收标准格式

每个功能的验收标准用 Given-When-Then 描述：

```
Given（前置条件）：用户已登录，且有管理员权限
When（操作）：点击「新增」按钮，填写表单，点击「提交」
Then（期望结果）：
  - 列表新增一条记录
  - 提示「新增成功」
  - 表单关闭
```

## 前端测试策略

- 组件测试：用 Vitest + Vue Test Utils
- E2E 测试：用 Playwright 覆盖关键流程
- 视觉回归：截图对比（可选）

## 后端测试策略

- 单元测试：Jest，覆盖 service 层业务逻辑
- 集成测试：supertest，覆盖所有 API 端点
- 数据库测试：使用测试数据库，不 Mock

## 测试数据管理

- 测试数据独立，不污染生产数据
- 每个测试用例执行前重置状态
- 使用工厂函数生成测试数据，不硬编码

## 测试完成检查清单

- [ ] 所有新功能有对应测试
- [ ] 覆盖率 ≥ 80%
- [ ] 边界值和异常场景已覆盖
- [ ] 权限控制已验证
- [ ] CI 流水线全部通过
- [ ] 无跳过的测试（`skip`/`todo`）
