## 二、项目概述

### 2.1 项目背景

PM能源集团内蒙古分公司目前管理下属 8 座油库的一次物流（从炼厂到油库运输）业务，涉及十几家炼厂（每年动态变化）和 20 余家承运商（不固定）。当前一次物流管理主要采用纸质单据方式，存在以下问题：

1. **过程不透明**：炼厂发运车辆数量、油库实际接卸车辆数量，全过程缺乏实时可视化监控，管理层无法掌握实时运营状态。
2. **信息滞后**：油库待卸车辆情况、接卸进度无法实时反馈，无法精准掌握各油库实时运营状态，影响调度决策。
3. **车辆积压风险**：油库待卸情况及接卸能力无法及时推送给承运商、炼厂和司机，易造成车辆积压，增加运营成本。
4. **数据孤岛**：地磅系统、液位仪系统、车辆运营平台等独立运行，数据无法联动，统计分析困难。
5. **效率低下**：手工填写纸质运单效率低、易出错，信息不标准化，后续统计追溯困难。

为提升一次物流运营效率、降低车辆等待时间、增强数据透明度，拟在现有协同管理工作平台（OA 系统）的作业安全管控平台基础上，新增一次物流信息化管理模块，实现从电子运单创建、发运、在途监控到公路收油的全流程数字化管理。

### 2.2 项目建设目标

本项目旨在建设一套安全、高效、透明的一次物流信息化管理平台，实现以下目标：

1. **全流程数字化**：实现从运单创建到接卸完成的全流程线上化，替代纸质单据，运单录入时间缩短 70%。
2. **数据自动采集**：集成地磅系统（多厂商适配）、液位仪系统（多厂商适配），自动获取重量、库存数据，过磅数据自动采集率达到 80% 以上。
3. **实时可视化**：车辆位置实时可视，油库接卸进度实时展示。
4. **移动化办公**：司机、承运商、炼厂人员通过现有 OA 系统移动端完成运单录入、状态查看、到库报到等操作，外部用户使用手机号加密码登录。
5. **数据联动**：库存数据、接卸数据、运单数据实时联动，支持日报、周报、月报及深度分析，辅助运营决策。

### 2.3 项目建设范围

**业务范围：**
- 内蒙古分公司下属 8 座油库的一次物流（公路收油）业务
- 十几家炼厂（动态变化）的发运管理
- 20 余家承运商（动态变化）的运输管理
- 电子运单、发运、在途、到库、接卸全流程

**系统范围：**
- 基础数据管理（炼厂、承运商、车辆、司机、油库、油品等）
- 电子运单管理
- 发运管理（含地磅集成）
- 在途监控（含多平台车辆定位适配）
- 公路收油（含地磅集成、液位仪集成）
- 报表中心（运单报表，含日/周/月/自定义粒度切换 + 多图表）
- 系统配置（操作日志）
- 企业与权限管理（部门、用户、角色、菜单权限）

**不包含的功能：**
- 二次物流（油库到加油站）管理
- 财务结算系统
- 炼厂内部生产管理

### 3.5 功能详细设计

#### 3.5.1 基础数据管理

**（一）功能需求概述**

基础数据管理是一次物流信息化平台的数据底座，负责维护炼厂、承运商、车辆、司机、油库、油罐等主数据。平台各业务模块（电子运单、在途监控、公路收油）均依赖本模块的主数据运转，因此本模块的数据准确性和完整性直接影响全平台业务的正常开展。

**（二）功能设计**

##### 3.5.1.1 炼厂管理

炼厂管理采用左右分栏布局：左侧为炼厂列表，右侧根据左侧选中项展示炼厂详情和炼厂人员列表。

**左侧：炼厂列表**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 炼厂名称 | 文本 | - |

支持按炼厂名称搜索。列表顶部有"全部炼厂"选项，选中时右侧展示所有炼厂人员汇总列表。
列表为空时显示：暂无炼厂数据。

**左侧操作**

- **导出炼厂**：点击"导出"，将当前炼厂列表导出为电子表格文件。导出成功提示"导出成功，文件已开始下载"。

**炼厂新增/编辑表单**

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |

提交成功：新增炼厂成功。提交失败：保留填写内容，显示具体错误原因。

**右侧：炼厂详情 + 炼厂人员**

| 状态 | 状态标签 | 已激活 / 已停用 |

**业务规则**

- 炼厂编码由系统自动生成，全局唯一，不可手动修改。
- 炼厂名称在系统内全局唯一。新增/编辑时若重复，阻止提交，提示"炼厂【XXX】已存在，请更换名称"。
- 统一社会信用代码须符合标准格式（18位），格式不符时阻止提交，提示"统一社会信用代码格式不正确，请重新输入"。
- 炼厂停用后，历史数据保留，系统不允许以该炼厂新建运单；已在途运单正常完成，不受影响。停用确认文案："停用后该炼厂将无法新建运单，确定停用吗？"
- 炼厂人员手机号全局唯一，同时作为登录账号。新增/编辑时若重复，阻止提交，提示"该手机号已被使用，请更换"。
- 导入文件格式不符时，提示"文件格式不正确，请使用系统提供的模板"。

**边界与异常**

- **空数据**：炼厂列表无数据时，显示"暂无炼厂数据"；人员列表无数据时，显示"暂无炼厂人员数据"。
- **权限不足**：新增、编辑、启用/停用、导入、导出按钮对无权限角色完全隐藏（不渲染，非禁用）。
- **统一社会信用代码格式错误**：提示"统一社会信用代码格式不正确，请重新输入"。

- 司机删除时，若司机当前绑定车辆或存在进行中的运单，阻止删除，提示"该司机存在关联数据，无法删除，请先解除绑定"。
- 所属承运商仅可选择状态为"启用"的承运商。
- 未配置运营平台的车辆，在途监控中定位方式标记为"手动定位"，由司机通过移动端手动上报位置。
- 承运商信息变更（名称、统一社会信用代码、联系人、联系电话、邮箱）时系统自动记录变更记录，含变更时间、变更人、变更前后内容。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无承运商数据 / 暂无车辆数据 / 暂无司机数据"。
- **权限不足**：新增、编辑、启用/停用、删除、绑定司机、导入、导出按钮对无权限角色完全隐藏（不渲染，非禁用）。
- **导入文件格式错误**：提示"文件格式不正确，请使用系统提供的模板"。

##### 3.5.1.3 库站管理

库站管理（即 V1.2 SRS 中的"油库与库容管理"）采用左右布局：左侧按分公司分组的组织架构树展示所有油库；右侧根据选中节点展示该油库的基本信息卡片和油罐列表。同页可发起油品目录维护。

**左侧：库站组织树**

按所属分公司动态分组，每个分公司节点下展开该分公司管辖的油库节点。库站节点 hover 时隐藏状态标签，显示启用/停用按钮。

| 字段类型 | 说明 |
|---------|------|
| 分公司分组 | 由库站的"所属分公司"字段自动聚合 |
| 库站节点 | 显示库站名称 + 状态标签（启用 / 停用 / 暂停接卸），hover 显示启停按钮 |

**右侧：库站基本信息卡片（选中具体库站时显示）**

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 库站名称 | 文本输入 | ≤100字 | 是 | - | 全局唯一 |
| 所属分公司 | 文本 | - | 是 | - | 用于左侧树形分组 |
| 联系人 | 文本输入 | ≤50字 | 是 | - | - |
| 联系电话 | 文本输入 | 11位 | 是 | - | - |
| 地址 | 文本输入 | ≤200字 | 否 | - | - |
| 液位仪型号 | 单选 | - | 否 | 手工录入 | 选择液位仪适配器，未配置时显示"手工录入" |
| 液位仪同步频率 | 数字输入 | - | 否 | 5 | 单位：分钟，仅在选择液位仪适配器后生效 |

卡片右上角"编辑"按钮可修改基本信息；卡片右上角设置图标进入油品目录维护弹窗。

**右侧：油罐列表**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 序号 | 数字 | 表格自动编号 |
| 罐号 | 文本 | 同一库站内唯一 |
| 油品种类 | 文本 | 来源于油品目录中已启用的油品 |
| 设计容量(吨) | 数字 | 保留2位小数 |
| 当前库存(吨) | 数字 | 实时库存，来源于液位仪自动同步或手工录入 |
| 空容(吨) | 数字 | 系统实时计算：设计容量 - 当前库存 |
| 数据来源 | 状态标签 | 液位仪自动同步 / 手工录入 |
| 操作 | 操作按钮 | 编辑、手工修正、删除（无权限时隐藏） |

列表为空时显示："暂无油罐数据"。选中"全部库站"或未选中具体库站时，油罐列表为空。

**油罐新增/编辑表单**

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 罐号 | 文本输入 | ≤20字 | 是 | - | 同一库站内唯一 |
| 油品种类 | 单选 | - | 是 | - | 来源于油品目录中已启用的油品 |
| 设计容量 | 数字输入 | - | 是 | - | 正数，保留2位小数，单位：吨 |

提交成功：新增油罐成功。提交失败：保留填写内容，显示具体错误原因。

**油罐操作**

- **编辑**：弹出编辑窗口，自动填入当前数据，表单字段同新增。提交成功：编辑油罐成功。
- **手工修正**：弹出修正窗口，填写修正后的库存值并提交审批。提交后提示"库存修正申请已提交，待审批后生效"。审批通过后库存数据生效，数据来源标记为"手工录入"。
- **删除**：弹出确认"确定删除该油罐吗？"。若油罐存在历史接卸记录，阻止删除，提示"该油罐存在历史接卸记录，无法删除"。

**油品目录管理**

通过右侧卡片右上角设置图标进入油品目录弹窗，维护系统支持的油品种类（用于油罐和运单的油品选择）。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 油品名称 | 文本输入 | ≤50字 | 是 | - | 全局唯一，如：92号汽油、0号柴油 |
| 油品编码 | 文本输入 | ≤20字 | 是 | - | 全局唯一，如：GA92、DI00 |
| 状态 | 单选 | - | 是 | 启用 | 启用 / 停用 |

**业务规则**

- 库站名称在系统内全局唯一。新增/编辑时若重复，阻止提交，提示"油库【XXX】已存在，请更换名称"。
- 罐号在同一库站内唯一。新增/编辑时若重复，阻止提交，提示"罐号【XXX】在本油库已存在，请更换"。
- 油品名称、油品编码均在系统内全局唯一。重复时阻止提交，提示"油品【XXX】已存在"。
- 空容由系统实时计算（空容 = 设计容量 - 当前库存），不存储不可手动填写。
- 液位仪数据按配置频率（默认每5分钟）自动同步；液位仪离线时改为手工录入，提示"液位仪连接异常，请手工录入库存数据"。
- 手工修正库存数据须经审批后方可生效，提交后提示"库存修正申请已提交，待审批后生效"。
- 库站状态枚举为 active（启用）、inactive（停用）、paused（暂停接卸）三种。"暂停接卸"状态下系统自动拒绝向该库站分配新运单，提示"该油库当前暂停接卸，无法分配新运单"。
- 库存数据出现异常突变时，系统触发库存预警，自动推送给省/分公司管理人员和油库主任。

**边界与异常**

- **空数据**：库站树为空时显示"暂无库站数据"；油罐列表为空时显示"暂无油罐数据"。
- **权限不足**：编辑、启用/停用/暂停接卸、油罐新增/编辑/删除/手工修正、油品目录维护按钮对无权限角色完全隐藏（不渲染，非禁用）。
- **液位仪离线**：系统改为手工录入模式，库存数据来源标记为"手工录入"，提示"液位仪连接异常，请手工录入库存数据"。

**（三）业务逻辑关联**

- 炼厂数据被电子运单管理引用，作为运单的发货方信息来源；炼厂停用后，电子运单管理中无法选择该炼厂新建运单。
- 承运商数据被电子运单管理引用；承运商停用后，其关联账号无法登录平台，且无法新建运单。
- 车辆数据被电子运单管理和在途监控引用；车辆的运营平台配置供在途监控模块获取车辆位置信息。
- 司机数据被电子运单管理引用，司机手机号同时作为司机移动端登录账号。
- 库站、油罐、液位仪数据被公路收油引用；库站状态为"暂停接卸"时，无法向该库站分配新运单。
- 油品目录数据被油罐管理和电子运单管理引用，油罐的油品种类和运单的油品仅可选择油品目录中已启用的油品。
- 库存异常突变时系统记录日志供管理人员查阅。

#### 3.5.2 电子运单管理

**（一）功能需求概述**

电子运单管理是物流全流程数字化的起点，替代纸质运单，承运商或炼厂人员通过系统创建电子运单，系统自动生成唯一运单号与二维码，并在运输全流程中实时跟踪运单状态。该功能为发运管理、在途监控、公路收油及报表中心提供核心数据支撑。

**（二）功能设计**

**查询/筛选功能**

支持以下条件筛选，条件之间为"与"关系，点击搜索按钮触发查询：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 日期范围 | 日期范围 | 按创建时间选择开始和结束日期 |
| 运单号 | 文本输入 | 支持模糊查询 |
| 运单状态 | 单选 | 枚举值：草稿 / 待发运 / 在途 / 到库 / 卸车中 / 已完成 / 已取消 |
| 炼厂 | 单选（可搜索） | 来源于基础数据管理中已启用的炼厂，filterable 模式 |
| 承运商 | 单选（可搜索） | 来源于基础数据管理中已启用的承运商，filterable 模式 |
| 目的油库 | 单选（可搜索） | 来源于基础数据管理中已录入的油库，filterable 模式 |
| 车牌号 | 文本输入 | 支持模糊查询 |
| 油品 | 文本输入 | 支持模糊查询 |

**列表功能**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 运单号 | 文本 | 左固定列。系统自动生成，草稿状态下显示"-" |
| 状态 | 状态标签 | 草稿 / 待发运 / 在途 / 到库 / 卸车中 / 已完成 / 已取消，light 填充无边框 |
| 炼厂 | 文本 | - |
| 目的油库 | 文本 | - |
| 承运商 | 文本 | - |
| 车牌号 | 文本 | - |
| 司机 | 文本 | - |
| 油品 | 文本 | - |
| 计划重量(吨) | 数字 | 右对齐，保留2位小数 |
| 计划发货时间 | 日期时间 | 草稿状态下无值显示"-" |
| 实际发货时间 | 日期时间 | 装车确认前显示"-" |
| 创建人 | 文本 | - |
| 创建时间 | 日期时间 | - |
| 操作 | 操作按钮 | 右固定列。详情、编辑（草稿状态）、取消（待发运状态），无权限时隐藏 |

分页默认每页10条，支持切换每页数量。列表为空时显示："暂无运单数据"。

**详情查看**

点击"详情"，打开运单详情面板，展示运单完整信息。内容分四个区域：

**地图区**（仅在途、到库、卸车中、已完成状态展示）：
- 起点标记：显示"发"标签 + 炼厂名
- 终点标记：显示"收"标签 + 油库名
- 路径：在途状态区分已走路段和未走路段；非在途状态显示单色路径
- 在途车辆：显示车辆位置标记 + 速度和预计到达时间
- 叠加显示：状态标签 + 车牌号

**运输进度**：待发运 → 在途 → 到库 → 待接卸 → 已完成，共 5 步。高亮当前节点，草稿和已取消状态不进入进度流。

**主信息卡**：展示发→收路线、油品、计划总量、承运商、司机、车牌号、创建人、计划发货时间、实际发货时间、装车总量（有值才显示）、出厂过磅、入库过磅、实际接卸、创建时间。

**物流动态**：状态流转记录倒序展示，每条含操作描述、操作时间、操作人。

**编辑功能**

点击"编辑"，自动填入当前记录数据，表单字段同新增运单。编辑入口仅在运单状态为"草稿"时显示，"待发运"及之后状态下编辑入口隐藏。
提交成功：系统自动生成运单号与二维码，运单状态变为"待发运"。
提交失败：保留填写内容，显示具体错误原因。

**取消运单功能**

点击"取消运单"，弹出确认"确定要取消该运单吗？取消后不可恢复"。
取消成功：提示"运单已取消"，运单状态变为"已取消"。
取消入口仅在运单状态为"待发运"时显示，其他状态下隐藏。

**批量导入功能**

点击"批量导入"，下载系统提供的标准 Excel 模板，按模板填写后上传文件。系统逐行校验，校验通过的行创建草稿运单，校验失败的行输出失败明细。
导入完成后提示："导入完成，成功X条，失败Y条，请下载失败明细查看原因"。
文件格式不符时提示："文件格式不正确，请使用系统提供的Excel模板"。

**导出功能**

点击"导出"，将当前筛选结果导出为 Excel 文件。数据量较大时异步生成，完成后通知下载，提示："正在生成导出文件，完成后将通知您下载"。

**业务规则**

- **运单号自动生成**：运单提交时，系统自动生成运单号，格式为 YM + 8位日期（YYYYMMDD）+ 4位当日流水序号（从0001起），全局唯一，不可手动修改。草稿状态下不生成运单号。
- **必填字段校验**：炼厂、承运商、车辆/车牌号、司机、油品、计划重量、目的油库、计划发货时间任一为空时，阻止提交，提示"请填写【字段名】"。
- **运单提交后不可直接编辑**：运单提交后状态变为"待发运"，编辑入口隐藏。如需修改，须走变更流程，提示"运单已提交，如需修改请发起变更申请"。
- **运单状态自动流转**：运单状态由系统根据各环节操作自动变更，无需手动切换。流转规则：提交→待发运；装车确认→在途；到库报到→到库；卸油开始→卸车中；卸油确认→已完成；发运前取消→已取消。
- **取消运单限制**：仅"待发运"状态的运单可执行取消操作，其他状态下取消入口隐藏。在途状态下尝试取消时，提示"运单已在途，无法取消"。
- **停用炼厂/承运商限制**：炼厂或承运商被停用后，新建运单时不可选择该炼厂或承运商，提示"该炼厂/承运商已停用，无法创建运单"。
- **数据权限控制**：内部用户按组织层级查看权限范围内的所有运单；外部用户仅可查看与自身关联的运单。无权限时提示"您无权查看该运单"，相关操作入口对无权限角色完全隐藏（不渲染，非禁用）。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无运单数据"。
- **筛选结果为空**：筛选后无匹配记录时，显示"未找到符合条件的运单"。
- **权限不足**：编辑、取消、导出等操作按钮对无权限角色完全隐藏（不渲染，非禁用）；无权限查看运单详情时，提示"您无权查看该运单"。
- **批量导入格式错误**：上传文件格式不符时，提示"文件格式不正确，请使用系统提供的Excel模板"，阻止导入。
- **在途状态取消**：运单处于"在途"及之后状态时，取消入口隐藏；若通过其他路径触发，提示"运单已在途，无法取消"。

**（三）业务逻辑关联**

- 炼厂、承运商、车辆、司机、油库、油品数据均来源于基础数据管理，需基础数据管理先完成配置并启用相应数据，运单创建功能方可正常使用。
- 承运商或炼厂在基础数据管理中被停用后，已有关联运单不受影响，但不可再新建关联该炼厂/承运商的运单。
- 处于"待发运"状态的运单被发运管理引用，发运管理中的装车确认操作将触发运单状态自动流转至"在途"。
- 处于"在途"状态的运单被在途监控引用，用于实时跟踪车辆位置与运输进度。
- 处于"到库"和"卸车中"状态的运单被公路收油引用，到库报到及卸油操作将触发运单状态自动流转。
- 处于"已完成"状态的运单被报表中心引用，作为运输完成数据的统计来源。

#### 3.5.3 在途监控

**（一）功能需求概述**

在途监控提供调度人员实时掌握"在途车辆位置"和"油库排队情况"的全屏地图大屏。点击侧栏菜单时在浏览器新标签打开，全屏渲染，便于在调度大屏上长时间值守。两种监控视角通过顶部分段切换器切换。

**（二）功能设计**

**入口与全屏渲染**

通过菜单"电子运单管理 → 在途监控"进入，新标签页打开，全屏展示，专为大屏值守场景优化。

**顶部品牌通栏**

| 区域 | 内容 |
|------|------|
| 左侧 | 品牌标题"一次物流信息管理模块 - 在途监控大屏" |
| 右侧 | 「返回系统」按钮 + 全屏切换 + 刷新 |

**模式切换器**

支持两种监控模式切换：
- **车辆监控**：默认模式，展示所有在线车辆位置
- **油库排队**：展示所有油库的排队情况

切换模式时清空地图标记、重置选中状态、关闭详情面板，重新加载对应模式数据。

**车辆监控模式 - 车辆列表**

| 区域 | 内容 |
|------|------|
| 顶部统计 | 在线车辆数 + 总车辆数 |
| 搜索框 | 支持车牌号 / 运单号 / 司机姓名实时过滤 |
| 车辆列表 | 每项显示：车牌 + 在线/离线状态 + 运单号 · 司机名 + 起终路线（炼厂 → 油库） |

列表为空时显示："暂无在线车辆"。

**车辆监控模式 - 地图标记**

地图展示所有车辆位置。点击单个车辆或列表项后：

1. 隐藏其他车辆标记，仅保留选中车辆
2. 显示起点标记（"发" + 炼厂名）和终点标记（"收" + 油库名）
3. 在途车辆显示双色路径（已走路段和未走路段区分显示）
4. 车辆标记按行驶方向旋转
5. 显示速度/预计到达信息（当前速度 + 预计送达时间 + 「查看运单」入口）
6. 点击「查看运单」打开运单详情面板

非在途车辆（已到库/卸车中）选中时，显示整条单色路径，不显示速度信息。

取消选中时，恢复所有车辆标记显示。

**油库排队模式 - 油库列表**

| 区域 | 内容 |
|------|------|
| 顶部统计 | 排队车辆总数 + 在用油库数 |
| 油库列表 | 每项显示：油库名 + 排队状态（"排队 N 车" / "空闲"）+ 最早到库时间 |

**油库排队模式 - 地图标记**

| 元素 | 说明 |
|-----|------|
| 油库标记 | 仓库图标 |
| 排队状态 | 显示排队车辆数或空闲状态 |
| 油库名标签 | 显示油库全名 |

**油库排队模式 - 选中油库**

点击地图油库标记或列表项后：

1. 地图自动定位到该油库
2. 打开详情面板，展示：
   - 顶部：油库名 + 关闭按钮
   - 汇总：待卸车数量 + 当前状态（繁忙/空闲）
   - 排队详情：每条显示序号 + 车牌 + 油品 + 司机 + 到库时间
   - 点击单条排队记录可查看对应运单详情

关闭详情面板或切换模式时，面板收起。

**业务规则**

- **在线判定**：运单状态为"在途"的车辆标记为在线；状态为"到库"或"卸车中"的车辆也出现在监控列表，但标记为离线，仍显示路线（单色灰）。
- **排队队列**：状态为"到库"的运单按目的油库自动归入对应队列，排队序号按到库登记时间顺序自动编号。
- **位置数据刷新**：车辆位置每 5 分钟内至少刷新一次；超过 5 分钟未收到位置数据的车辆，自动标记为"离线"。
- **定位方式**：未配置运营平台的车辆，由司机通过移动端手动上报位置，定位方式标记为"手动定位"；其余车辆通过承运商运营平台获取，标记为"平台定位"。
- **路线规划失败**：路线规划失败时静默处理，不显示路线，不报错提示。
- **数据权限**：外部用户（承运商、炼厂、司机）仅可查看与自身关联的在途车辆和油库排队。
- **刷新防抖**：刷新按钮 400ms 内不可重复触发。

**边界与异常**

- **空数据**：车辆列表为空时显示"暂无在线车辆"；油库列表为空时显示"暂无油库数据"；油库排队为空时详情面板显示"该油库当前无车辆排队"。
- **车辆离线**：车辆标记为"离线"，列表中状态徽章变灰，地图保留最后已知位置，速度/ETA 信息卡不显示。
- **路线规划接口异常**：静默忽略，不影响其他元素渲染。
- **运单号反查 ID 失败**：静默忽略，不打开运单详情抽屉。
- **window.close() 被浏览器拒绝**：100ms 后自动跳转到首页 `/`。

**（三）业务逻辑关联**

- 在途车辆数据来源于电子运单管理中状态为"在途/到库/卸车中"的运单。
- 油库排队数据来源于电子运单管理中状态为"到库"的运单，按目的油库归类。
- 车辆运营平台配置来源于基础数据管理-承运商管理-车辆 Tab，需完成平台配置后方可获取平台定位。
- 运单详情抽屉与运单台账共用同一组件（WaybillDetailDrawer），通过 waybillId 联动。

#### 3.5.4 公路收油

**（一）功能需求概述**

公路收油负责车辆到达油库后的全流程操作，覆盖到库登记、入库过磅、卸油作业和接卸进度反馈四个环节。本功能解决待卸情况无法实时反馈、地磅数据手工录入效率低的问题，支持司机自助报到与油库操作员代登记两种方式，并将接卸进度实时推送给承运商、炼厂及在途司机。

**（二）功能设计**

**到库登记功能**

司机到达油库后，可通过扫描运单二维码自助报到，或由油库操作员代为登记。登记成功后系统自动分配排队序号，运单状态变更为"到库"。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 运单号 | 文本输入 | - | 是 | - | 扫码自动获取或手工输入 |
| 到库时间 | 只读 | - | - | 系统自动记录 | 登记时刻由系统自动记录，不可手动修改 |
| 排队序号 | 只读 | - | - | 系统自动分配 | 按到库登记顺序自动分配，不可手动修改 |

登记成功：提示"到库登记成功，排队序号：X"。

**入库过磅功能**

完成到库登记后，对车辆进行入库过磅。油库已配置地磅时，入库过磅重量由地磅自动获取；未配置地磅或地磅离线时，由操作员手工录入。系统自动计算磅差及磅差率。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 运单号 | 只读 | - | - | 自动带入 | 来源于到库登记，不可修改 |
| 车牌号 | 只读 | - | - | 自动带入 | 来源于到库登记，不可修改 |
| 出厂过磅重量 | 只读 | - | - | 自动带入 | 来源于发运管理中的出厂过磅记录；无出厂过磅数据时显示"-" |
| 入库过磅重量 | 数字输入 | 保留2位小数 | 是 | - | 单位：吨；已配置地磅时自动获取，未配置或地磅离线时手工录入 |
| 数据来源标记 | 只读 | - | - | 系统自动标记 | 枚举值：地磅自动获取 / 手工录入 |
| 磅差 | 只读 | - | - | 系统自动计算 | 计算方式：入库过磅重量 - 出厂过磅重量；无出厂数据时显示"-" |
| 磅差率 | 只读 | - | - | 系统自动计算 | 计算方式：磅差 ÷ 出厂过磅重量 × 100%；无出厂数据时显示"-" |
| 过磅单照片 | 图片上传 | 单张≤10MB，最多5张 | 否 | - | 支持上传过磅单凭证照片 |

提交成功：提示"过磅记录已保存，请继续完成卸油作业"。
提交失败：保留填写内容，显示具体错误原因。

**卸油作业功能**

入库过磅完成后，操作员发起卸油作业。支持单罐卸油和多罐分卸，各罐卸油量之和应等于入库过磅重量（允许合理误差）。

| 字段名称 | 类型 | 长度限制 | 必填 | 默认值 | 说明 |
|---------|------|---------|------|-------|------|
| 运单号 | 只读 | - | - | 自动带入 | 来源于入库过磅，不可修改 |
| 车牌号 | 只读 | - | - | 自动带入 | 来源于入库过磅，不可修改 |
| 入库过磅重量 | 只读 | - | - | 自动带入 | 来源于入库过磅记录，不可修改 |
| 卸油罐号 | 单选或多选 | - | 是 | - | 来源于基础数据管理中该油库已录入的油罐；支持多罐分卸 |
| 各罐卸油量 | 数字输入 | 保留2位小数 | 是 | - | 单位：吨；多罐分卸时每罐单独填写 |
| 卸油开始时间 | 只读 | - | - | 系统自动记录 | 点击开始卸油时由系统自动记录，不可修改 |
| 卸油结束时间 | 只读 | - | - | 系统自动记录 | 点击完成卸油时由系统自动记录，不可修改 |
| 卸油照片 | 图片上传 | - | 否 | - | 支持上传卸油过程凭证照片 |

卸油完成确认后：提示"卸油完成，运单已结束"。

**接卸进度反馈功能**

系统实时向承运商、炼厂及在途司机推送当前油库的接卸进度，使相关方无需主动查询即可掌握油库接卸状态。

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 待卸车辆数 | 数字 | 当前油库排队等待接卸的车辆数量（辆） |
| 剩余空容 | 数字 | 当前油库各油罐剩余可接收容量之和（吨），来源于基础数据管理-油库库容管理 |
| 预计等待时间 | 文本 | 系统根据当前待卸车辆数和历史接卸效率估算（小时） |
| 最近推送时间 | 日期时间 | 最近一次向承运商、炼厂及在途司机推送的时间 |

**业务规则**

- 司机到库后完成登记，运单状态变为"到库"，系统自动分配排队序号；提示"到库登记成功，排队序号：X"。
- 油库已配置地磅时，入库过磅重量由地磅自动获取，数据来源标记为"地磅自动获取"；未配置地磅时，由操作员手工录入，数据来源标记为"手工录入"；地磅离线时改为手工录入，提示"地磅设备当前离线，请手工录入重量数据"。
- 磅差超过阈值（默认±3‰）时触发磅差预警，需人工确认后方可继续卸油流程；提示"磅差【X‰】超出允许范围（±3‰），请确认是否继续卸油"。
- 卸油开始后运单状态变为"卸车中"；卸油完成确认后运单状态变为"已完成"；提示"卸油完成，运单已结束"。
- 卸油完成后，系统同步更新该油库的库存数据，库容数据实时刷新。
- 多罐分卸时，各罐卸油量之和应等于入库过磅重量（允许合理误差）。
- 接卸进度（待卸车辆数、剩余空容、预计等待时间）实时推送给承运商、炼厂及在途司机。
- 重量字段输入为0或负数时阻止提交，提示"重量必须大于0"。
- 图片上传超出大小限制时阻止上传，提示"图片大小不能超过10MB，请重新选择"。

**边界与异常**

- **空数据**：待卸列表无数据时，显示"暂无待卸车辆"。
- **地磅离线**：改为手工录入，数据来源标记为"手工录入"，提示"地磅设备当前离线，请手工录入重量数据"。
- **磅差超阈值**：提示"磅差【X‰】超出允许范围（±3‰），请确认是否继续卸油"，需人工确认后方可继续。
- **油罐空容不足**：所选油罐空容不足时阻止提交，提示"所选油罐空容不足，请重新选择"。
- **权限不足**：到库登记、入库过磅、卸油作业等操作按钮对无权限角色完全隐藏（不渲染，非禁用）。

**（三）业务逻辑关联**

- 到库登记依赖电子运单管理中状态为"在途"的运单，运单须已存在方可发起登记。
- 入库过磅中的出厂过磅重量来源于发运管理中的出厂过磅记录，用于磅差计算；无出厂过磅数据时磅差及磅差率均显示"-"。
- 卸油罐号来源于基础数据管理中该油库已录入的油罐；地磅适配器配置及磅差阈值由系统内置默认值控制。
- 卸油完成后，系统同步更新基础数据管理中对应油库的库容数据。
- 接卸进度实时推送依赖基础数据管理中油库的液位仪配置。

#### 3.5.5 报表中心 - 运单报表

**（一）功能需求概述**

运单报表合并 V1.2 中独立的"日报表 / 周报表 / 月报表 / 接卸数据分析"四个二级页为单一报表中心入口，通过粒度切换器（按日 / 按周 / 按月 / 自定义）切换不同时间口径，并以"KPI 指标卡 + 趋势图 + 占比图 + 排名图 + 磅差统计 + 明细表"组合，提供一次物流运营数据的多维度统计分析。本功能为只读统计展示，不支持手动新增或编辑报表数据。数据权限按角色范围自动限定。

**（二）功能设计**

**（二）功能设计**

**入口**

侧栏菜单"报表中心 → 运单报表"。报表中心一级菜单仅保留"运单报表"一个二级页（V1.2 中的"接卸数据分析"已并入运单报表的图表展示）。

|---------|------|------|

| 日期 | 日期选择器（粒度=按日时） | 选择单日，格式 YYYY-MM-DD |
| 周 | 周选择器（粒度=按周时） | 格式：YYYY 第 ww 周 |
| 月份 | 月份选择器（粒度=按月时） | 格式 YYYY-MM |
| 日期范围 | 日期范围选择器（粒度=自定义时） | 格式 YYYY-MM-DD 至 YYYY-MM-DD |
| 目的油库 | 单选（可搜索） | 来源于基础数据中已录入的油库 |

**KPI 指标卡**

**油库接卸占比**

按油库或油品维度展示接卸量占比分布。支持通过切换器在"按油库"和"按油品"两个维度间切换，数据集独立。

**承运商排名 / 炼厂排名**

| 图表 | 说明 |
|-----|------|
| 承运商排名 | 按发运量降序排列，显示数值（吨） |
| 炼厂排名 | 按发运量降序排列，显示数值（吨） |

**磅差统计**

展示出厂过磅量、入库过磅量（双柱对比）和磅差率（折线，单位‰）的组合对比。

**明细数据表**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 序号 | 数字 | 表格自动编号 |
| 周期 | 文本 | 按粒度显示：日期 / 周次 / 月份 / 自定义周期 |
| 目的油库 | 文本 | - |
| 运单数(单) | 数字 | 右对齐 |
| 总接卸量(吨) | 数字 | 右对齐，保留2位小数 |
| 完成(单) | 数字 | 右对齐 |
| 取消(单) | 数字 | 右对齐 |
| 平均接卸(吨) | 数字 | 右对齐，保留2位小数 |

列表为空时显示："所选时间范围内暂无数据"。

**业务规则**

- 切换粒度时联动重置日期字段、KPI 指标、所有图表、明细表；查询失败时保留筛选条件，提示"查询失败，请稍后重试"。
- 油库占比支持切换"按油库 / 按油品"两个维度，数据集独立。
- 日报表每日 24:00 自动生成，周报表每周一自动生成，月报表每月 1 日自动生成。
- 导出当前筛选条件下的数据为电子表格文件（.xlsx）或 PDF 文件；数据量较大时异步生成，提示"正在生成导出文件，完成后将通知您下载"。
- 数据权限按角色范围自动限定：省公司管理人员可查看全部 8 座油库数据；分公司管理人员仅可查看辖区内油库数据；油库角色仅可查看本库数据；承运商和炼厂角色仅可查看与自身关联的数据。

**边界与异常**

- **空数据**：所选时间范围内无数据时，明细表显示"所选时间范围内暂无数据"；图表显示空状态占位。
- **权限不足**：无权限角色对应的菜单入口完全隐藏（不渲染，非禁用）。
- **导出文件生成中**：数据量较大时，导出操作触发后提示"正在生成导出文件，完成后将通知您下载"。

**（三）业务逻辑关联**

- 总运单数、总接卸量、单均接卸量来源于电子运单管理中已完成的运单数据，按粒度聚合。
- 完成率 = 已完成运单数 / 总运单数。
- 接卸量趋势按粒度聚合后展示。
- 油库占比来源于按目的油库或油品聚合的接卸量。
- 承运商/炼厂排名来源于已完成运单的发运量按承运商/炼厂聚合后降序。

- "接卸数据分析"（V1.2 单独章节）的高峰时段分析、车辆等待时间分布、接卸瓶颈识别等深度分析后续可作为图表扩展直接挂载本页面，不再单独成页。

#### 3.5.6 系统管理

**（一）功能需求概述**

系统管理模块为平台提供用户账号、角色权限及操作日志的统一管理能力。管理员通过本模块完成内外部用户的审核激活与状态管控、角色与权限的配置分配，以及全平台操作行为的追溯查询，保障平台安全合规运行。

**（二）功能设计**

---

**3.5.6.1 用户管理**

**查询/筛选功能**

支持以下条件筛选，条件之间为"与"关系，点击搜索按钮触发查询：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 用户姓名 | 文本输入 | 支持模糊搜索 |
| 手机号 | 文本输入 | 精确匹配 |
| 用户类型 | 单选 | 枚举值：内部用户 / 外部用户 |
| 用户状态 | 单选 | 枚举值：待审核 / 已激活 / 已停用 |
| 所属单位 | 文本输入 | 支持模糊搜索，外部用户填写所属承运商或炼厂名称 |

**列表功能**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 用户姓名 | 文本 | - |
| 手机号 | 文本 | - |
| 用户类型 | 文本 | 内部用户 / 外部用户 |
| 所属单位 | 文本 | 所属承运商或炼厂名称；内部用户显示所属部门 |
| 用户状态 | 状态标签 | 待审核 / 已激活 / 已停用 |
| 注册时间 | 文本 | - |
| 操作 | 操作按钮 | 审核激活（待审核状态显示）、停用（已激活状态显示）、启用（已停用状态显示）；无权限时隐藏 |

分页默认每页 10 条，支持切换每页数量。列表为空时显示：暂无用户数据。

**用户来源说明**

内部用户由企业协同管理平台自动同步至本系统，不在本系统单独注册，无新增入口。外部用户（承运商管理人员、炼厂操作人员、司机）通过手机号在移动端自助注册，注册后状态为"待审核"，由管理员在本页面完成审核激活。

**用户基本信息说明**

用户基本信息（姓名、手机号、所属承运商/炼厂等）由注册时填写确定，注册后不支持编辑。管理员仅可对用户执行审核激活、停用、启用操作。

**审核激活功能**

点击"审核激活"，弹出确认"确认激活该用户吗？"，确认后将用户状态变更为"已激活"。
激活成功：显示"用户已激活"。

**停用/启用功能**

点击"停用"，弹出确认"确认停用该用户吗？停用后该用户将无法登录和操作。"，确认后将用户状态变更为"已停用"。
停用成功：显示"用户已停用"。

点击"启用"，弹出确认"确认启用该用户吗？"，确认后将用户状态变更为"已激活"。
启用成功：显示"用户已启用"。

**业务规则**

- 手机号在系统内全局唯一。外部用户注册时若手机号已被使用，阻止注册，提示"该手机号已被使用，请更换"。
- 用户状态变更为"已停用"后，该用户在所有功能模块中的操作权限立即失效；已在途的操作（如已提交的运单、已发起的接卸流程）不受影响，继续按原流程执行。
- 已停用用户尝试登录时，系统拒绝登录，提示"该账号已停用，请联系管理员"。
- 外部用户注册成功后，移动端显示"注册申请已提交，请等待管理员审核"，用户进入待审核状态，审核激活前无法使用系统功能。

**边界与异常**

- **空数据**：列表无数据时，显示"暂无用户数据"。
- **权限不足**：审核激活、停用、启用按钮对无权限角色完全隐藏（不渲染，非禁用）。

## 四、技术方案

### 4.1 技术架构

本系统在现有 OA 系统（作业安全管控平台）基础上新增一次物流信息化管理模块，复用现有 OA 系统的前端框架、后端服务和数据库基础设施，不单独部署新系统。

**终端架构：**
- PC 端：OA 系统 Web 端扩展，新增一次物流管理菜单模块
- APP 端：OA 系统 APP 扩展，内外部用户统一使用同一 APP，外部用户使用通过账号密码方式登录平台

**集成适配层：**
三类外部系统集成均采用适配器模式，定义统一标准接口，各厂商/平台实现各自适配器，新增厂商只需实现标准接口，不改动业务代码：
- 地磅适配器：支持多厂商地磅系统接入，无地磅时改为手工录入
- 液位仪适配器：支持多厂商液位仪系统接入，无液位仪时改为手工录入
- 车辆定位适配器：支持多平台车辆运营系统接入，无定位平台时由司机通过移动端手动上报位置

### 4.2 数据交互

- 前后端数据交互采用 HTTPS 加密传输
- 地磅、液位仪数据通过适配器定时拉取，刷新频率可配置（默认每5分钟）
- 车辆位置数据通过适配器定时拉取，刷新频率不超过5分钟
- 预警通知通过消息队列异步推送，支持站内消息和短信双通道

### 4.3 外部系统集成

| 集成系统 | 集成方式 | 说明 |
|---------|---------|------|
| OA 系统 | 用户体系同步 | 内部用户账号和组织架构从 OA 系统同步 |
| 地磅系统（多厂商） | 适配器模式 | 各炼厂/油库按实际厂商配置对应适配器 |
| 液位仪系统（多厂商） | 适配器模式 | 各油库按实际厂商配置对应适配器 |
| 车辆运营平台（多平台） | 适配器模式 | 各承运商按实际平台配置对应适配器 |

### 4.4 安全方案

- 身份认证：内部用户使用 OA 系统统一认证，外部用户使用手机号加密码认证
- 数据传输：全程 HTTPS 加密
- 数据存储：敏感数据加密存储
- 访问控制：基于角色的权限管理（RBAC），功能权限细化到按钮级别，数据权限按组织层级隔离

## 五、非功能需求

### 5.1 性能要求

- 系统响应时间：≤3 秒（95% 请求）
- 并发用户数：支持 ≥500 人同时在线
- 车辆位置数据刷新：≤5 分钟
- 液位仪数据同步：≤5 分钟

- 用户认证：OA 系统统一认证（内部用户）、手机号加密码（外部用户）
- 数据传输：HTTPS 加密
- 数据存储：敏感数据加密存储
- 访问控制：基于角色的权限管理（RBAC）

### 5.3 可用性要求

- 系统可用性：≥99.5%

- OA 系统集成：用户体系、组织架构同步
