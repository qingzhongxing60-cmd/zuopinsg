# 测试用例文档规范

## 文档结构

```
一、文档信息
二、修订历史
三、引言（编写目的、测试范围、参考文档）
四、测试环境
五、测试用例编号规范
六、模块测试用例
七、测试总结
```

## 卡片式用例表格规范

每个用例一个独立表格，格式：

```markdown
**表 X.X.X-N [用例名称]**

| 用例编号 | [编号] | 用例名称 | [名称] |
|:---:|:---|:---:|:---|
| **用例描述** | [描述本用例的测试目标] | ^ | ^ |
| **前置条件** | [执行用例所需的前置状态] | ^ | ^ |
| **步骤** | **操作步骤** | ^ | **预期结果** |
| 1 | [操作步骤描述] | ^ | [预期结果描述] |
| 2 | [操作步骤描述] | ^ | [预期结果描述] |
```

规范说明：
- `^` 表示水平合并，与同行左侧相邻单元格合并
- 用例描述/前置条件行：后三列合并，内容写在第二列
- 步骤数据行：第三列与第二列合并（操作步骤占两列宽），第四列写预期结果
- 预期结果必须具体明确，不能写"显示正确"、"操作成功"

## 各类测试覆盖场景

**业务功能测试：**
- 正常操作流程（新增、查询、编辑、删除）
- 数据查询与筛选
- 状态流转（审批、启用/停用）
- 导入导出（如有）、批量操作（如有）

**异常测试：**
- 必填字段为空、字段超出长度限制、输入非法格式
- 删除/编辑已关联或已使用的数据
- 重复提交（防止并发重复数据）

**性能测试：**
- 列表页面加载时间（大数据量下）
- 查询/筛选响应时间、文件导入/导出处理时间

**权限测试：**
- 无权限账号访问功能
- 未登录状态访问系统
- 不同角色看到的菜单和按钮差异

## 测试总结模板

```markdown
## 七、测试总结

本测试用例文档共包含 **[N] 个功能模块**，**[N] 个测试用例**。

### 7.1 测试用例统计

| 模块 | 测试用例数量 |
|:---:|:---:|
| [模块名1] | [数量] |
| **合计** | **[总数]** |

### 7.2 测试覆盖率

- **业务功能测试**：覆盖所有核心业务流程
- **异常测试**：覆盖必填字段为空、格式校验、关联数据删除限制等
- **权限测试**：覆盖各角色的访问权限及菜单可见性
- **性能测试**：覆盖列表加载、查询响应等关键操作

### 7.3 后续工作

1. **测试执行**：逐条执行测试，记录结果及缺陷
2. **缺陷跟踪**：对问题分级（致命/严重/一般/建议），提交缺陷报告并跟进修复
3. **回归测试**：缺陷修复后执行回归测试
4. **验收测试**：协同客户进行 UAT
5. **文档归档**：更新文档状态为"已完成"，归档测试报告
```

## Word 导出

```bash
bash .claude/skills/common/export-word.sh <markdown文件路径> test-cases
```

导出的 Word 文档自动处理：
- `^` 标记的单元格自动水平合并
- 4 列测试用例表格自动设置固定列宽
- 粗体行（用例描述/前置条件/步骤标题）自动填充蓝色背景
- 步骤编号列自动居中对齐
