# 作品时光个人作品集系统设计方案

## 一、项目概述

在现有三端工程基础上，构建一个展示个人作品的作品集系统。后台用于管理作品、原型、拆解文章及站点内容，前端面向访客展示作品集与个人成长轨迹。

**三端职责：**

| 端 | 目录 | 技术栈 | 本次工作 |
|----|------|--------|---------|
| 后端服务 | `server/` | NestJS + Prisma + MySQL | 新增作品集业务模型与接口 |
| 管理后台 | `admin/` | Vue3 + Element Plus | 新增作品集管理模块，界面层简化为单管理员 |
| 展示站点 | `website/` | Vue3 + TailwindCSS | 替换现有营销模板为作品集展示前端 |

**目标用户：**
- 管理员（作品集主人）：通过后台维护作品、原型、拆解、站点内容
- 访客：通过展示站点浏览作品、原型、成长轨迹与个人介绍

**权限策略**：后端保留登录认证，后台界面不暴露组织/角色/权限管理页面，仅保留单一管理员登录入口。

## 二、功能范围

**后台管理（admin）：**
1. 作品管理 — 作品的新增、编辑、删除、发布、设为精选，富文本详情、封面、分类
2. 原型管理 — 在作品下管理多版本（V1/V2/V3），每版本多张图片上传与排序
3. 拆解管理 — 在作品下管理拆解文章（富文本）
4. 分类管理 — 分类的新增、编辑、删除，作为前端筛选项的动态来源
5. 时间轴管理 — 成长轨迹年份节点的新增、编辑、删除、排序
6. 关于我管理 — 个人简介、技能数据、简历内容维护
7. 首页配置 — 首页展示文案、精选作品选取
8. 数据统计 — 作品访问量排行展示

**展示前端（website）：**
1. 首页 — 展示入口、导航、精选作品
2. 作品列表 — 按分类筛选浏览作品
3. 作品详情 — 原型展示（图片放大、版本切换）与拆解文章
4. 时间轴 — 年份节点成长轨迹
5. 关于我 — 技能展示与简历

## 三、核心模块设计

### 3.1 作品管理

作品是系统核心实体。管理员可新增、编辑、删除作品，每个作品包含标题、唯一标识、简介、封面图、富文本详情、所属分类、精选标记、发布状态、访问量、排序值、发布时间。

- 发布状态：草稿 / 已发布。草稿不在展示端公开。
- 精选标记：标记后用于首页精选展示。
- 作品在编辑页内集中管理其原型版本与拆解文章。

### 3.2 原型管理（隶属作品）

每个作品下可维护多个原型版本（如 V1、V2、V3），用于迭代展示。每个版本包含版本名、标题、排序值，版本下可上传多张图片，每张图片含说明文字与排序值。展示端支持版本切换与图片放大查看。

### 3.3 拆解管理（隶属作品）

每个作品下可维护多篇拆解文章，记录项目拆解内容。文章包含标题、富文本正文、发布状态、排序值。草稿不在展示端公开。

### 3.4 分类管理

管理员维护作品分类（如 AI、医疗、外呼、CRM），包含分类名、唯一标识、排序值。分类作为作品的归属维度，也是展示端作品列表筛选项的动态来源。删除分类时若有作品引用，阻止删除并提示。

### 3.5 时间轴管理

管理员维护成长轨迹年份节点，每个节点包含年份、标题、描述、排序值。展示端按排序展示成长轨迹。

### 3.6 关于我管理

管理员维护个人介绍信息（单条），包含个人简介、头像、富文本简历；并维护技能列表，每项技能含名称、评分值、排序值，作为展示端能力展示的数据来源。

### 3.7 首页配置

管理员维护首页展示文案（展示标语、副标题等），以及精选作品的选取，控制展示端首页呈现内容。

### 3.8 数据统计

记录作品详情页的访问量。访问展示端作品详情时，对应作品访问量累加（同一访客短时间内重复访问只计一次）。后台展示作品访问量排行。

## 四、数据结构与流向

**核心数据实体：**

| 实体 | 关键字段 | 关系 |
|------|---------|------|
| 分类 | 名称、唯一标识、排序值 | 一个分类对应多个作品 |
| 作品 | 标题、唯一标识、简介、封面、富文本详情、所属分类、精选标记、发布状态、访问量、排序值、发布时间 | 隶属一个分类；包含多个原型版本、多篇拆解 |
| 原型版本 | 版本名、标题、排序值 | 隶属一个作品；包含多张图片 |
| 原型图片 | 图片地址、说明、排序值 | 隶属一个原型版本 |
| 拆解文章 | 标题、富文本正文、发布状态、排序值 | 隶属一个作品 |
| 时间轴节点 | 年份、标题、描述、排序值 | 独立维护 |
| 技能 | 名称、评分值、排序值 | 独立维护，关于我展示用 |
| 个人介绍 | 简介、头像、富文本简历 | 单条记录 |
| 首页配置 | 展示文案、精选作品选取 | 单条记录 |

**数据流向：**
- 管理员在后台维护各实体数据，保存至后端
- 展示端通过公开只读方式获取已发布数据；草稿状态数据不返回
- 访客访问作品详情时，触发作品访问量累加
- 展示端作品列表筛选项来源于分类管理数据

## 五、边界情况处理

- **访问量防刷**：作品访问量累加按访客来源去重，同一来源短时间内重复访问只计一次
- **草稿隔离**：草稿状态的作品与拆解不在展示端公开，仅后台可见
- **分类删除保护**：删除分类时若有作品引用，阻止删除并提示先迁移作品
- **原型版本顺序**：版本与图片均按排序值展示，展示端版本切换按此顺序
- **空状态**：列表、详情、时间轴等无数据时展示友好空态，不报错
- **图片展示**：原型图支持大图预览，列表使用缩略图并按需加载
- **唯一标识**：作品唯一标识用于展示端访问路径，需校验唯一，重复时提示
