# 详细设计章节标准结构

## 每个功能模块（3.5.x）的完整结构

```
#### 3.5.x.x 功能名称

**（一）功能需求概述**
[1-3句话说明该功能的目的和价值，不写实现方式]

**（二）功能设计**

**[子功能1]功能**
[功能说明段落]

[字段表格（如有）]

**业务规则**
[触发条件 + 影响范围 + 提示文案原文]

**边界与异常**
[异常场景 + 处理方式 + 提示文案原文]

**（三）业务逻辑关联**
[依赖哪些其他模块的数据，跨模块影响说明]

**（四）用户操作流程图**
![](images/[功能名]-operation-flow.png)

**（五）功能时序图**
![](images/[功能名]-sequence.png)
```

## 各部分写法规范

### （一）功能需求概述

- 1-3 句话，说明"这个功能解决什么问题"、"谁在用"、"核心价值是什么"
- 不写页面布局，不写技术实现

```
✅ 示例：
设备档案管理功能用于维护厂区所有设备的基本信息，支持按分类、单位、状态多维度查询，
为巡检计划配置和设备状态监控提供基础数据支撑。
```

---

### （二）功能设计 — 筛选功能

```markdown
**查询/筛选功能**

支持以下条件筛选，条件之间为"与"关系，点击搜索按钮触发查询：

| 筛选字段 | 类型 | 说明 |
|---------|------|------|
| 字段名 | 文本输入 | 支持模糊搜索 |
| 字段名 | 单选 | 枚举值：A/B/C |
| 字段名 | 层级选择 | 来源于[模块]中已启用的[数据] |
| 字段名 | 日期范围 | 选择开始和结束日期 |
```

---

### （二）功能设计 — 列表功能

```markdown
**列表功能**

| 列名 | 字段类型 | 说明 |
|-----|---------|------|
| 字段名 | 文本 | - |
| 字段名 | 状态标签 | 启用（绿色）/停用（灰色） |
| 字段名 | 图片+文本 | 左侧缩略图，点击可预览大图；无图显示占位图 |
| 字段名 | 文本 | 无值显示"-" |
| 操作 | 操作按钮 | 编辑、删除（无权限时隐藏） |

分页默认每页10条，支持切换每页数量。
支持批量勾选，勾选后操作栏显示批量删除按钮（含已选数量）。
列表为空时显示：暂无[数据名]数据。
```

```markdown
点击"新增"进入新建[数据名]页面（或弹出编辑窗口）。

**编辑功能**

点击"编辑"，自动填入当前记录数据，表单字段同新增。

```

### （二）功能设计 — 业务规则

**必须包含三要素：触发条件 + 影响范围 + 提示文案原文**

```markdown
**业务规则**

- [字段名]在系统内全局唯一。新增/编辑时若重复，阻止提交，提示"[提示文案原文]"。
- [状态]变更为[值]后，[跨模块影响说明，用业务语言描述]。
- 删除[数据名]时，若存在关联[数据名]，阻止删除，提示"[提示文案原文]"。
```

### （二）功能设计 — 边界与异常

```markdown
**边界与异常**

- **空数据**：列表无数据时，显示"[提示文案原文]"
- **权限不足**：[操作]按钮对无权限角色完全隐藏（不渲染，非禁用）
- **[其他异常场景]**：[处理方式和提示文案]
```

---

### （三）业务逻辑关联

```markdown
**（三）业务逻辑关联**

- [字段名]数据来源于[模块名]，需要[模块名]先完成配置
- [本模块]的[状态/数据]变更后，影响[其他模块]的[具体行为]
- [本模块]被[其他模块]引用时，删除需先解除关联
```

## 章节完整性检查

每个功能模块写完后，对照以下清单：

- [ ] （一）功能需求概述：有，且不含技术词汇
- [ ] 筛选功能：有字段表格，来源描述用业务语言
- [ ] 列表功能：有字段表格，含分页说明、空数据提示
- [ ] 新增/编辑功能：有字段表格，含必填/长度/默认值/提示文案
- [ ] 删除功能：有确认文案原文，有关联保护说明（如有）
- [ ] 业务规则：有触发条件 + 影响范围 + 提示文案原文
- [ ] 边界与异常：有空数据、权限不足场景
- [ ] （三）业务逻辑关联：有跨模块依赖说明
- [ ] （四）（五）流程图占位：有图片引用

## 3.3 页面功能清单 — 功能描述列写法（L1 规则）

功能描述列遵循 L1 概览层规则（见 prd-language.md 内容分级规则）：动词 + 对象 + 条件

| 正确 | 错误 |
|------|------|
| "查看四项运营指标统计" | "4 张 KPI 卡，顶部 3px 彩色细条" |
| "按时间维度查看接卸量变化趋势" | "折线 + 蓝色面积渐变图" |
| "按油库或油品维度查看占比" | "圆环饼图，可切换维度" |
| "按发运量查看承运商排名" | "横向柱状图，蓝色渐变" |
| "查看运单详细信息" | "右侧滑出面板，宽 640px" |

**原则：** 功能描述只回答"用户能做什么"，不回答"界面长什么样"。

## （二）功能设计 — 视觉与交互规格（可选，L3）

当功能有特殊视觉要求时，在子功能行为描述之后用独立标题包裹：

```markdown
**[子功能名] - 视觉与交互规格**

| 元素 | 规格 |
|------|------|
| KPI 卡片 | 顶部 3px 彩色细条区分类别 |
| 详情面板 | 右侧滑出，宽 640px |
| 趋势图 | 折线 + 面积渐变填充 |
```

**使用规则：**
- 此子节不受 prd-language.md L1/L2 禁止词约束
- 仅在功能有明确的视觉设计要求时添加，不是每个功能都需要
- 必须物理独立于功能行为描述，不可混写在同一段落中
- 位置：放在对应子功能的「边界与异常」之后、下一个子功能之前

## 多端系统章节规范

系统包含 PC 端和移动端时，必须遵守以下规范：

### 3.1 总体功能架构 — 多端写法

必须分端描述菜单结构，不可混写：

```
**PC 端（管理后台）**

- [一级菜单]（一级菜单）
  - [二级菜单]（二级菜单）

**移动端**

- 首页（底部导航 Tab）
  - [子功能]
- 工作台（底部导航 Tab）
  - [子功能]
- 消息（底部导航 Tab）
  - [子功能]
- 我的（底部导航 Tab）
  - [子功能]
```

**移动端 Tab 命名规范：**
- 底部导航 Tab 必须使用通用 App 导航结构：**首页、工作台、消息、我的**
- 禁止按业务功能命名 Tab（如"炼厂操作"、"在途/到库"等），这不符合 App 导航惯例
- 业务操作功能统一放在"工作台" Tab 下，按角色权限过滤显示

### 3.3 页面功能清单 — 多端写法

表格必须包含"终端"列（5列），PC 端和移动端分开列出：

```
| 终端 | 一级菜单 | 二级菜单 | 子功能 | 功能描述 |
|-----|---------|---------|-------|---------|
| PC端 | ... | ... | ... | ... |
| ^ | ... | ... | ... | ... |
| 移动端 | 首页 | - | 待办事项 | [描述] |
| ^ | ^ | - | 数据概览 | [描述] |
| ^ | 工作台 | - | [业务功能] | [描述] |
| ^ | 消息 | - | 通知列表 | [描述] |
| ^ | 我的 | - | 个人信息 | [描述] |
```

- 业务操作功能全部放在"工作台"Tab 下，用 `**功能名称**` 小标题展开，不拆独立章节
- 工作台的业务规则中必须说明各角色可见的功能入口（权限过滤逻辑）
- 移动端字段表格同样需要六列（字段名称/类型/长度限制/必填/默认值/说明）
- 图片命名：`mobile-[tab英文名]-operation-flow.png` 和 `mobile-[tab英文名]-sequence.png`
