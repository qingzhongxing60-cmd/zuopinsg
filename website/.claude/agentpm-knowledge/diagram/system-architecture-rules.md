# 系统架构图绘图规范

> 适用于：系统架构图、功能架构图、组织结构图。
> 前置依赖：必须先读取 `common-rules.md` 通用规范。
> 模板目录：`examples/system-architecture/`

---

## 一、布局模式

架构图使用**水平分层 + 容器嵌套**布局，不使用泳道（swimlane）。

### 1.1 分层结构

```
┌─────────────────────────────────────────────┐
│  系统标题（text 节点，居中，大字号加粗）       │
├─────────────────────────────────────────────┤
│  [层标签]  │  层背景容器（圆角矩形）          │
│            │    ├── 模块标题（小圆角色块）     │
│            │    └── 子功能节点（白底网格）     │
├─────────────────────────────────────────────┤
│  [层标签]  │  层背景容器                      │
│            │    └── 模块节点（单行排列）       │
└─────────────────────────────────────────────┘
```

### 1.2 层间关系表达

**禁止使用箭头连接层与层。** 架构图靠视觉层次（从上到下 = 从前端到底层）表达依赖关系，不需要 edge。

若确实需要表达数据流向，使用极细的虚线（strokeWidth=1;dashed=1;strokeColor=#cccccc），且不能穿过任何容器。

## 二、节点类型

### 2.1 层标签（左侧）

每层左侧有一个胶囊形标签，标识该层名称：

```
style="rounded=1;whiteSpace=wrap;html=1;fillColor={层色};strokeColor=none;fontSize=11;fontStyle=1;fontColor=#ffffff;arcSize=50;"
尺寸：width=80, height=28
```

### 2.2 层背景容器

每层有一个浅色背景矩形，包裹该层所有模块：

```
style="rounded=1;whiteSpace=wrap;html=1;fillColor={浅色};strokeColor={边框色};arcSize=3;"
```

### 2.3 模块标题

核心业务层中，每个模块有一个深色标题条：

```
style="rounded=1;whiteSpace=wrap;html=1;fillColor={深色};strokeColor=none;fontSize=11;fontStyle=1;fontColor=#ffffff;arcSize=10;"
```

### 2.4 子功能节点

模块标题下方的具体功能点：

```
style="rounded=1;whiteSpace=wrap;html=1;fillColor=#ffffff;strokeColor={边框色};fontSize=11;"
```

### 2.5 带描述的模块节点

基础数据层、外部系统层使用 HTML 内联标题+描述：

```xml
value="&lt;b&gt;模块名&lt;/b&gt;&lt;br&gt;&lt;font style=&quot;font-size:10px;color:#666&quot;&gt;描述文字&lt;/font&gt;"
```

**禁止用两个独立节点叠放来实现标题+描述效果，必须合并为单个 cell 的 HTML value。**

## 三、配色方案

| 层级 | 层标签色 | 背景色 | 边框色 | 模块标题色 | 语义 |
|------|---------|--------|--------|-----------|------|
| 用户终端 | #1565c0 | #e3f2fd | #90caf9 | — | 蓝色系 |
| 核心业务 | #e65100 | #fff8e1 | #ffe082 | #fb8c00 | 橙色系 |
| 基础数据 | #2e7d32 | #e8f5e9 | #a5d6a7 | — | 绿色系 |
| 系统支撑 | #4527a0 | #ede7f6 | #b39ddb | — | 紫色系 |
| 外部集成 | #b71c1c | #fce4ec | #ef9a9a | — | 红色系 |

可根据实际层数增减，但同一图中不超过 6 种色系。

## 四、间距规范

| 项目 | 最小值 |
|------|--------|
| 层与层垂直间距 | 15px |
| 层标签与层背景水平间距 | 15px |
| 模块节点间水平间距 | 15px |
| 子功能节点间水平间距 | 10px |
| 子功能节点间垂直间距 | 8px |
| 节点距层背景边缘 | 12px |
| 图例距最后一层 | 20px |

---

## 五、禁止行为

1. **禁止使用 emoji** — draw.io 渲染 API 不支持 emoji，会显示为方块
2. **禁止使用 swimlane 作为层容器** — swimlane 会产生内部标题栏和折叠箭头，用普通 rounded 矩形
3. **禁止文字节点叠放** — 标题和描述必须在同一个 cell 的 value 中用 HTML 换行
4. **禁止层间箭头** — 架构图不需要 edge，靠视觉层次表达
5. **禁止使用 shape=cylinder3 等特殊形状** — 统一用圆角矩形，保持视觉一致性
6. **禁止超过 6 层** — 层数过多会导致图表过长，合并相近层级

## 六、自检清单

生成 XML 后、输出前，逐项检查：

- [ ] 无 emoji 字符
- [ ] 无 swimlane 样式（搜索 "swimlane" 应为 0 结果）
- [ ] 无 edge 节点（搜索 "edge=" 应为 0 结果，除非有明确的虚线数据流需求）
- [ ] 所有多行文字使用 HTML `<b>` + `<br>` + `<font>` 内联，无独立叠放的 text 节点
- [ ] 每层背景容器宽度一致（对齐）
- [ ] 层标签垂直居中于对应层背景
- [ ] 子功能节点网格对齐（同列 X 相同，同行 Y 相同）
- [ ] 图例存在且颜色与实际层色一致
