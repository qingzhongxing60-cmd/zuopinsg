# 标注策略规范

## 核心原则

**标注的目的是让开发看着页面就能获取完整需求信息——需求说要有什么，代码里是怎么实现的，两者对照才能让开发一眼看懂。**

标注内容来自两个来源：
- **需求文档**：字段定义、业务规则、提示文案（"做什么"）
- **代码实现**：实际的交互细节、联动逻辑、组件结构（"怎么实现的"，验证需求是否落地）

---

## 多页面功能模块处理规范

一个功能模块（如"工单台账"）通常包含多个独立路由页面，**每个页面都需要单独标注，生成独立的 JSON 文件**。

### 识别方法

拿到功能名称后，先扫描对应的视图目录：

```bash
ls {PROJECT_PATH}/src/views/{ModuleDir}/
```

常见文件结构：

| 文件名 | 对应页面类型 | 标注 JSON 命名示例 |
|--------|------------|-----------------|
| `index.vue` | 列表页 | `rectification-workorder.json` |
| `Detail.vue` | 详情页 | `rectification-workorder-detail.json` |
| `Create.vue` | 新增页（独立路由） | `rectification-workorder-create.json` |
| `Edit.vue` | 编辑页（独立路由） | `rectification-workorder-edit.json` |

### 处理原则

- **不要只标注 index.vue 就结束**，必须把目录下所有独立路由页面都纳入标注任务
- 详情页、新增页、编辑页各自有独立的字段和交互逻辑，需要单独分析需求文档对应章节
- 多步表单页（如巡检计划新增）也是独立路由，按多步表单页策略单独标注
- 弹窗/抽屉内的表单不是独立路由，用 `container: modal/drawer` 处理，不单独生成 JSON

### 路由确认

扫描视图目录后，还需确认路由配置中是否有对应路由：

```bash
grep -n "Detail\|Create\|Edit" {PROJECT_PATH}/src/router/modules/{module}.ts
```

有路由配置的页面文件才需要单独标注；纯子组件（无路由）不需要。

**先读 `package.json` 确定框架，再读组件文件判断页面类型。**

### 框架识别

| 框架组合 | package.json 特征 | 组件文件后缀 |
|---------|-----------------|------------|
| Vue 3 + Ant Design Vue | `vue` + `ant-design-vue` | `.vue` |
| Vue 3 + Element Plus | `vue` + `element-plus` | `.vue` |
| Vue 3 + Vant（移动端） | `vue` + `vant` | `.vue` |
| React + Ant Design | `react` + `antd` | `.tsx` / `.jsx` |
| React + Arco Design | `react` + `@arco-design/web-react` | `.tsx` / `.jsx` |
| Next.js + Ant Design | `next` + `antd` | `.tsx`（app/ 或 pages/） |

### 页面类型识别（读组件文件后判断）

不依赖固定 class 名，而是根据**组件结构特征**判断：

| 页面类型 | Vue 识别特征 | React 识别特征 |
|---------|------------|--------------|
| **列表页** | 有筛选表单区 + `a-table`/`el-table` + 分页 | 有筛选区 + `Table` 组件 + `Pagination` |
| **详情页** | 有返回按钮 + `a-descriptions`/`el-descriptions` + 无分页 | 有返回按钮 + `Descriptions` + 无分页 |
| **多步表单页** | 有 `a-steps`/`el-steps` + 分步内容区 | 有 `Steps` 组件 + 分步内容区 |
| **混合页** | 列表 + 详情/树在同一页 | 同左 |
| **纯表单页** | 有 `a-form`/`el-form` + 提交按钮 + 无表格 | 有 `Form` + 提交按钮 + 无 `Table` |

## 列表页标注策略

每个功能区域对应一个标注，区域内所有字段说明合并写入 content。

| 功能区域 | 标注标题 | category | 选择器策略 |
|---------|---------|---------|-----------|
| 左侧树/分类树 | xxx 树 | field | 树容器 |
| 筛选区 | 筛选区字段 | filter | 筛选区容器（一般是筛选卡片整体） |
| 数据列表 | 列表字段 | field | 表格本身（`.ant-table`、`.el-table` 等由 class 注入包起来） |
| 操作栏单个按钮 | 新增按钮 / 导入按钮 / 导出按钮 / 批量删除按钮 | action | 按钮本身（逐个注入 class） |
| 新增/编辑弹窗或抽屉 | 新增/编辑表单字段 | field | 表单元素，`container: modal` / `drawer` |
| 导入弹窗 | 导入弹窗 | action | 弹窗内容区，`container: modal` |
| 操作列按钮（编辑/删除/绑定/启停） | 列表 - 编辑按钮 / 列表 - 删除按钮 | action | 按钮本身（逐个注入 class） |

**列表页特别说明：**
- **操作栏不单独标注整体**：新增/导入/导出/批量删除等按钮已经逐个单独标注，不要再给整个操作栏外层加一个 `annot-action-bar` 标注，否则范围覆盖全部按钮，视觉上多一个多余的标注点。
- 行内状态开关（`a-switch`）的切换规则合并到列表字段标注里，不单独占一个标注点。
- 跳转到详情页的入口（如点击任务名称）要在列表字段或操作列按钮标注里说明。
- 操作列对应的行为说明写在每个按钮自己的标注里（编辑/删除/绑定各一个），不要合并成一个"操作列说明"。

## 详情页标注策略

详情页的标注重点是：**展示了哪些字段、字段的含义、特殊渲染规则、操作按钮的业务逻辑**。

| 功能区域 | 标注标题 | category | 选择器策略 |
|---------|---------|---------|-----------|
| 页面头部（状态标签+操作按钮） | 详情页头部说明 | action | `.detail-header` |
| 基本信息区（a-descriptions） | 基本信息字段说明 | field | `.ant-descriptions` |
| 折叠面板区（a-collapse） | xxx记录说明 | field | `.ant-collapse` |
| 时间轴区（a-timeline） | 流转记录说明 | field | `.ant-timeline` |
| 操作按钮区（审核/整改/复核） | 操作说明 | action | 锚定到对应按钮 class |
| 照片/附件区 | 照片/附件说明 | field | 找容器 class 或加 class |

**详情页特别说明：**
- 状态标签要说明所有可能的状态值和对应颜色含义
- 折叠面板每个 panel 的展开内容（如检查清单结果）要在折叠面板标注里说明
- 时间轴的每个节点代表什么操作、由谁触发，要在流转记录标注里说明
- 异常高亮（如 `.row-abnormal` 红色背景行）要在对应字段标注里说明触发条件

---

## 多步表单页标注策略

多步表单页的标注重点是：**每步的字段、步骤间的校验规则、联动逻辑、提交结果**。

| 功能区域 | 标注标题 | category | 选择器策略 |
|---------|---------|---------|-----------|
| 步骤条 | 分步填写流程说明 | field | `.form-steps` |
| 工具栏（返回+步骤条+操作按钮） | 页面操作说明 | action | `.form-toolbar` |
| 第N步字段区 | 第N步：xxx字段说明 | field | `.form-steps`（复用，内容区分步骤） |

**多步表单页特别说明：**
- 步骤条标注说明总步数、每步名称、步骤间的校验规则（每步校验通过才能进入下一步）
- 每步的字段单独标注，不要把四步字段合并到一个标注里（太长，开发看不清楚）
- 联动字段（如切换巡检方式时清空另一种模式的数据）要在对应步骤标注里说明
- 提交成功/失败的结果说明放在工具栏标注里

## container 字段：弹窗/抽屉内的标注

每个标注项有一个可选的 `container` 字段，控制标注点渲染到哪个层：

| container 值 | 行为 | 适用场景 |
|-------------|------|---------|
| `page`（默认） | 渲染在页面层，弹窗/抽屉打开时自动隐藏 | 页面上的筛选区、列表、操作栏等 |
| `modal` | teleport 到弹窗容器，弹窗打开时才显示，关闭时自动消失 | 弹窗内的表单字段、操作说明 |
| `drawer` | teleport 到抽屉容器，抽屉打开时才显示，关闭时自动消失 | 抽屉内的表单字段、操作说明 |

**使用规则：**
- 标注弹窗内的内容时，设置 `"container": "modal"`，selector 指向弹窗内的元素（如 `.ant-modal-body`）
- 标注抽屉内的内容时，设置 `"container": "drawer"`，selector 指向抽屉内的元素（如 `.ant-drawer-body`）
- 不设置 container 或设置为 `page` 时，弹窗/抽屉打开时标注点自动隐藏，避免遮挡

## 选择器优先级决策树

每个标注点的定位按以下顺序执行，**selector 字段是提示而非硬约束**，找不到时自动降级，不会导致标注点消失：

```
第一步：用 selector 字段直接查 DOM
  → 找到 → 定位到该元素
  → 找不到 ↓

第二步：根据 category + title 动态推断候选元素
  filter   → 找筛选区容器（[class*="filter"]、[class*="search"]、form）
  action   → 找操作栏容器（[class*="action"]、[class*="toolbar"]）
  field    → 根据 title 关键词找表格/表单/详情/步骤条/折叠面板
  rule     → 就近挂在操作栏或表格上
  → 找到 → 定位到该元素（多个标注点自动错开偏移）
  → 找不到 → 保持上次位置，不消失，等下次重试
```

**因此 selector 字段的写法原则：**
- 写语义化 class（`.filter-card`、`.action-bar`）→ 最稳定，优先
- 写 UI 库渲染 class（`.ant-table`、`.el-table`）→ 次优，跨项目通用
- 写不出合适选择器时 → **直接留空字符串 `""`**，系统会根据 category+title 自动推断
- 不需要为了"有选择器"而写一个不准确的选择器

## 没有选择器时的处理流程

当目标元素没有合适的 CSS 选择器时，按以下顺序处理：

**第一步：检查是否有语义化 class**
- 读取对应的 Vue/React 组件文件
- 查找目标元素（按钮、容器等）是否有 `class` 属性

**第二步：没有 class → 在代码里加**
- 给触发按钮加语义化 class，命名规则：`btn-{动作}`，如 `btn-add`、`btn-import`、`btn-bind-checklist`
- 给容器加语义化 class，命名规则：`{区域}-card` 或 `{区域}-container`
- 加完 class 后，用新 class 作为选择器

**第三步：无法加 class（如第三方组件内部元素）→ 用 position**
- 只有在无法修改源码的情况下才使用 `type: position`
- 使用 position 时，在标注 content 里注明"需手动拖放到合适位置"

## 额外单独标注的情况

以下情况在区域标注之外额外单独标注：

| 情况 | 标注标题格式 | category |
|------|------------|---------|
| 字段有唯一性校验 | 业务规则：xxx 唯一 | rule |
| 状态变更有跨模块影响 | 业务规则：xxx 状态影响 | rule |
| 删除有关联限制 | 业务规则：删除限制 | rule |
| 有复杂联动逻辑 | 业务规则：xxx 联动 | rule |
| 状态流转规则 | 状态流转说明 | rule |
| 异常高亮/特殊渲染 | 业务规则：xxx 高亮 | rule |

---

## 禁止标注的内容（重要）

以下内容**不要**生成独立标注点，也不要写进其他标注的 content：

| 不标 | 原因 |
|------|------|
| 角色权限矩阵、"仅 XX 角色可见"、"无权限隐藏" | 权限由业务配置权限码控制，不是设计约束。标注只描述功能行为。 |
| "操作栏整体说明"（当各按钮已单独标注） | 整体范围覆盖所有按钮，和每个按钮标注重叠 → 视觉上多一个多余点。 |
| 确认弹窗文案、成功/失败提示 | 合并到触发该弹窗的操作按钮 content 里说明。 |
| 分页默认值、空数据提示文案 | 合并到列表字段标注里。 |
| "点击 X 按钮可以做 Y" 的重复陈述 | 标注本身就锚定在按钮上，直接写行为（"打开新增弹窗"）。 |

---

## 不需要单独标注的内容

以下内容合并到区域标注的 content 里说明，不单独占一个标注点：

- 分页默认值（合并到列表标注）
- 确认弹窗文案（合并到操作栏或删除标注）
- 成功/失败提示文案（合并到对应操作标注）
- 空数据提示文案（合并到列表标注）
- 行内状态开关的切换规则（合并到列表字段标注）

## 标注数量参考

| 页面类型 | 标注数量参考 |
|---------|------------|
| 标准列表页（含筛选、列表、新增/编辑、导入导出） | 7-10 个 |
| 详情页（含基本信息、折叠面板、时间轴、操作按钮） | 5-8 个 |
| 多步表单页（4步） | 6-8 个（步骤条1个 + 每步1个 + 工具栏1个 + 业务规则1-2个） |
