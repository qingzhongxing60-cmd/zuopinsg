## 写作前必读：禁止词汇速查

生成任何内容前，确认不包含以下词汇：

| 禁止 | 替换为 |
|------|-------|
| 弹窗、Modal、Drawer、抽屉 | 弹出编辑窗口、弹出确认、右侧编辑面板 |
| 数据源：/api/xxx | 来源于[模块]中已启用的[数据] |
| 树形下拉、TreeSelect | 层级选择 |
| 开关组件、Switch | 开关、状态切换 |
| 回填、回显 | 自动填入已有数据 |
| 步骤条、Steps 组件 | 分步填写（共N步） |
| Tag 形式展示 | 逐个展示，支持移除 |
| hover 显示、tooltip | 悬停显示 |
| 调用接口、API 请求 | 提交、查询、获取 |
| 推送通知 | 通知、提醒 |
| 前端过滤、不请求接口 | 在已加载数据中过滤 |
| a-xxx、el-xxx、van-xxx | 禁止出现任何 UI 组件名 |
| flex、grid、margin、px | 禁止出现任何 CSS 属性 |

## 二、项目概述

### 2.1 项目背景

**内容要求：**
- 描述行业现状和面临的挑战，说明项目建设的必要性
- 列出3-5个具体问题点，每个问题点加粗标题 + 具体描述
- 字数：300-500字

**示例：**
```
随着[行业]的快速发展，[业务场景]面临以下挑战：

1. **[问题标题]**：[具体描述，说明问题的影响和后果]
2. **[问题标题]**：[具体描述]
3. **[问题标题]**：[具体描述]

为解决上述问题，建设[项目名称]，实现[核心价值]，提升[管理目标]。
```

### 2.2 项目建设目标

**内容要求：**
- 列出3-5个具体、可衡量的目标
- 每个目标：加粗名称 + 详细说明（包含功能点和业务价值）
- 字数：每个目标80-150字

**示例：**
```
本项目旨在建设一套[系统类型]，实现以下目标：

1. **[目标名称]**：[详细说明目标内容、实现方式和预期效果]
2. **[目标名称]**：[详细说明]
```

### 2.3 项目建设范围

**内容要求：**
- 明确系统端（PC端、移动端等）
- 列出功能模块清单
- 说明数据范围
- 明确不包含的功能（如有）

**示例：**
```
本项目建设范围包括：

**系统端**：
- PC端管理后台（Web应用）
- 移动端H5（支持微信/浏览器访问）

**功能模块**：
- 模块一：功能说明
- 模块二：功能说明

**数据范围**：
- 数据类型一：数据说明

**不包含的功能**：
- 功能一：说明原因（预留接口，后续迭代实现）
```

**必须调用 diagram-generator 技能生成**，参考模板目录：`examples/bpmn/`，图片命名：`images/business-flow.png`

**内容要求：** 使用表格定义各类用户及其权限范围。

**示例：**
```
| 角色名称 | 角色描述 | 主要职责 | 权限范围 |
|---------|---------|---------|---------|
| 系统管理员 | 负责系统配置和用户管理 | 用户管理、权限配置 | 全部功能 |
```

## 三、项目建设内容

### 3.1 总体功能架构

**必须调用 diagram-generator 技能生成系统功能架构图**，图片命名：`images/system-architecture.png`

**多端架构说明（架构图下方补充）：**

若系统包含多个终端（PC 端、移动端等），必须分端描述菜单结构，不可混写。格式如下：

```
**PC 端（管理后台）**

- [一级菜单]（一级菜单）
  - [二级菜单]（二级菜单）

**移动端**

- [Tab 名称]（底部导航 Tab）

```

**单端菜单结构文字说明：**
- 只列出菜单中显示的功能，不列出详情页、编辑页等隐藏页面
- 标注层级：（一级菜单）、（二级菜单）
- 不标注权限说明（权限在 3.4 中统一说明）

**示例（单端）：**
```

  - 用户管理（二级菜单）
  - 角色管理（二级菜单）
```

### 3.2 需求功能清单

**内容要求：**
- 从业务视角描述系统需要满足的需求，不涉及具体页面和功能点
- 优先级：P0（核心必须）、P1（重要）、P2（一般）
- 必须覆盖所有核心业务需求

**示例：**
```
| 序号 | 业务需求 | 需求概述 | 优先级 |
|-----|---------|---------|-------|
| 1 | 用户身份认证 | 系统用户通过账号密码登录，不同角色拥有不同功能权限 | P0 |
| 2 | 设备档案管理 | 统一维护设备基本信息，支持多维度查询，作为巡检和整改的基础数据 | P1 |
```

### 3.3 页面功能清单

**内容要求：**
- **多端系统必须分端列表**：PC 端和移动端分开写，不可混在一张表里
- 5列表格：**终端**、一级菜单、二级菜单、子功能、功能描述
  - 终端列：PC端 / 移动端（相同终端使用 `^` 合并）
  - 没有二级菜单时，二级菜单列填 `-`
  - 相同的一级菜单、二级菜单使用 `^` 合并单元格
- **子功能必须拆到最小操作粒度，每个操作独立一行**
  - PC 端标准操作集：筛选、列表、新增、编辑、删除、导出、导入、启用/停用、详情查看
  - 移动端标准操作集：查看列表、查看详情、提交、确认、上传照片、扫码等
  - 特有操作：审核、分配权限、连接测试、发布、撤回等，各自独立一行
- **功能描述列遵循 L1 概览层规则**（见 prd-language.md 内容分级规则）：
  - 格式：动词 + 对象 + 条件（可选）
  - 禁止出现图表视觉类型、颜色、尺寸、布局位置等视觉描述

**示例（多端）：**
```
| 终端 | 一级菜单 | 二级菜单 | 子功能 | 功能描述 |
|-----|---------|---------|-------|---------|
| PC端 | 设备管理 | 设备档案 | 筛选 | 按设备名称、分类、状态筛选 |
| ^ | ^ | ^ | 列表 | 分页展示设备列表，支持导出 |
| ^ | ^ | ^ | 新增 | 填写设备基本信息，上传图片和附件 |
| ^ | 报表中心 | 运单报表 | 指标统计 | 查看总运单数、总接卸量、完成率、单均接卸量四项指标 |
| ^ | ^ | ^ | 趋势分析 | 按时间维度查看接卸量变化趋势 |
| ^ | ^ | ^ | 占比分析 | 按油库或油品维度查看占比分布 |
| ^ | ^ | ^ | 排名对比 | 按发运量查看承运商和炼厂排名 |
| ^ | 系统配置 | 用户管理 | 列表 | 查询用户列表 |
| 移动端 | 工作台 | - | 创建运单 | 填写运单信息并提交 |
| ^ | ^ | - | 运单管理 | 查看进行中和已完成的运单列表 |
| ^ | 消息 | - | 通知列表 | 查看系统通知和业务提醒 |
```

## 五、非功能需求

### 5.1 性能需求

- 响应时间：页面加载时间 < 3秒，接口响应时间 < 1秒
- 并发用户数：支持 XXX 并发用户
- 数据处理能力：每秒处理 XXX 条数据

### 5.2 安全需求

- 身份认证：账号密码登录，支持多因素认证
- 数据安全：敏感数据加密存储，传输使用 HTTPS
- 权限控制：基于角色的访问控制（RBAC）

### 5.3 可用性需求
